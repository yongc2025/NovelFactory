# Resume Workflow

Use this workflow when a long-form project continues after interruption, a new chat, or missing context.

## Goal

Recover enough project state to continue safely without rereading the entire novel.

## Required inputs

Prefer these files:

1. `project_manifest.md`
2. `latest_state.md`
3. current `chapter_table.md`
4. latest chapter `handoff.md`
5. relevant `knowledge_state.md`
6. relevant `foreshadowing_ledger.md`

If unavailable, reconstruct temporary state from the user's latest material.

## Resume steps

1. Identify project id and current stage.
2. Read latest state and latest handoff.
3. Recover confirmed canon.
4. Recover uncertain canon and mark as `待确认`.
5. Recover protagonist knowledge state.
6. Recover current emotional phase.
7. Recover active conflicts and promises.
8. Identify the next safe production action.
9. Continue from the closest stable state.

## Do not

- Do not assume missing facts are canon.
- Do not draft a chapter if current knowledge state is unknown.
- Do not resolve a major promise if payoff timing is unknown.
- Do not introduce high-level lore to fill gaps.

## Resume report template

```markdown
# Resume Report

## project_id

## recovered_stage

## confirmed_canon

## uncertain_facts

## protagonist_current_state

## active_conflicts

## active_promises

## continuity_risks

## next_safe_action
```
