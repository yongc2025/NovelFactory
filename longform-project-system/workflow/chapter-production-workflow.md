# Chapter Production Workflow

Use this workflow for every chapter in a long-form novel project.

## Inputs

Required or reconstructed:

- project bible
- 12-layer ledger
- current volume outline
- chapter table
- character bible
- knowledge-state ledger
- emotion-arc ledger
- foreshadowing ledger
- reader-promise ledger
- previous chapter handoff

## Workflow

```text
read state
-> verify four-question gate
-> build chapter card
-> split scene cards
-> draft prose
-> run editorial audit
-> update ledgers
-> write handoff
```

## Step 1: State recovery

Before planning the chapter, identify:

- current chapter number
- current volume goal
- protagonist's current external objective
- protagonist's current emotional phase
- protagonist's known, guessed, misunderstood, and forbidden knowledge
- unresolved promises
- active conflicts
- active foreshadowing

## Step 2: Four-question gate

Do not draft prose unless the chapter answers:

1. What does the protagonist know now?
2. Why must the protagonist act now?
3. How is new information learned on-page?
4. What changes by the end of the chapter?

## Step 3: Five-part chapter card

Every chapter card must include:

- chapter goal
- obstacle
- action
- change
- hook

A chapter is weak if it lacks goal, resistance, or change.

## Step 4: Scene cards

Split each chapter into 3-5 scenes by conflict unit, not by location.

Each scene card must define:

- location and time
- POV
- who wants what
- what blocks them
- stakes
- conflict mode
- turning point
- result
- cost
- new information
- emotion shift
- next-scene hook

## Step 5: Drafting

Draft prose only from approved scene cards.

Protect:

- canon
- character voice
- knowledge boundaries
- emotional phase
- information-release order
- reader promise pacing

## Step 6: Audit

After drafting, run an audit for:

- continuity
- knowledge leakage
- emotional jump
- unearned ability
- premature lore
- weak hook
- forgotten foreshadowing
- broken reader promise

## Step 7: Ledger update

Update affected ledgers:

- chapter table
- knowledge state
- emotion arc
- information release
- foreshadowing
- reader promise
- continuity
- revision log, if edits were made

## Step 8: Handoff

Write `handoff.md` for the chapter. The next session must be able to continue from it without re-reading the entire project.
