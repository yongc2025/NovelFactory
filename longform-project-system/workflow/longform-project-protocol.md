# Longform Project Protocol

Use this protocol when creating, continuing, revising, or managing a full-length novel or serialized fiction project.

## Core principle

Treat the novel as durable project state.

Do not produce floating content that is not assigned to a target file or project stage.

## Standard session flow

Every session follows this order:

1. Locate the project.
2. Read `project_manifest.md` and `latest_state.md`.
3. Identify the current stage.
4. Load the minimum required state files.
5. Execute exactly one primary production task.
6. Audit continuity and reader promise impact.
7. Update affected ledgers.
8. Write a handoff package.
9. Update `latest_state.md`.

## Project stages

```text
intake
foundation
worldbuilding
character_design
macro_outline
volume_outline
chapter_batch_outline
chapter_card
scene_cards
prose_draft
editorial_audit
revision
export
```

## Stage rules

### Intake

Collect or infer:

- title or working title
- genre
- target length
- target platform
- POV and tone
- reference works
- forbidden content or style limits

### Foundation

Establish the 12-layer system before expanding lore or drafting prose.

Required layers:

1. core proposition
2. reader promise
3. protagonist wound
4. protagonist desire
5. world pressure
6. conflict system
7. information release
8. character arc
9. chapter design
10. scene design
11. satisfaction design
12. revision audit

### Macro outline

Do not write a full chapter before the current volume goal and first 5-10 chapter beats exist.

### Chapter production

A chapter must pass:

1. four-question gate
2. five-part chapter card
3. scene conflict cards
4. prose draft
5. editorial audit
6. handoff update

## Four-question gate

Before chapter drafting, answer:

1. What does the protagonist know now?
2. Why must the protagonist act now?
3. How does new information appear on-page?
4. What changes in situation, relationship, knowledge, or emotion?

## Handoff requirement

Every output that changes the project must include a handoff package with:

- current stage
- confirmed canon updates
- knowledge-state updates
- emotion-arc updates
- information-release updates
- foreshadowing updates
- reader-promise updates
- continuity risks
- unresolved questions
- next recommended action

## Missing state policy

If required files are missing:

1. reconstruct temporary state from available materials,
2. mark uncertain facts as `待确认`,
3. continue only with reversible assumptions,
4. do not silently invent canon.
