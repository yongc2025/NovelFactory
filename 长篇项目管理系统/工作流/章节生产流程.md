# 章节生产流程

本流程用于长篇小说每一章的规划、正文、审校和状态更新。

## 输入文件

优先读取：

- 项目圣经
- 十二层创作台账
- 当前卷大纲
- 章节表
- 角色手册
- 人物认知状态
- 情绪线
- 伏笔台账
- 读者承诺台账
- 上一章交接包

## 流程

```text
读取状态
→ 确认本章位置
→ 回答四问门槛
→ 制作章节卡
→ 制作场景卡
→ 写正文草稿
→ 做审校
→ 更新台账
→ 写交接包
```

## 四问门槛

写正文前必须回答：

1. 主角现在知道什么？
2. 主角为什么现在必须行动？
3. 新信息如何在正文中出现？
4. 本章结束时局面、关系、认知或情绪有什么变化？

## 章节五要素

每章必须具备：

- 章节目标
- 阻碍
- 行动
- 变化
- 钩子

如果缺少目标、阻碍或变化，本章应先重做章节卡，不进入正文。

## 场景卡要求

场景不是地点，而是冲突单元。

每个重要场景至少记录：

- 地点与时间
- POV
- 谁想要什么
- 谁或什么阻止
- 双方筹码
- 转折点
- 场景结果
- 代价
- 新增信息
- 情绪变化
- 下一场钩子

## 正文草稿规则

正文只从已确认的场景卡生成。

必须保护：

- 既定项目事实
- 人物说话方式
- 人物认知边界
- 当前情绪阶段
- 信息释放顺序
- 读者承诺节奏

## 审校检查

正文完成后检查：

- 连续性是否破坏
- 人物是否知道过早
- 能力是否突然跳级
- 情绪是否跳跃
- 设定信息是否解释过多
- 章末钩子是否有效
- 伏笔是否新增、推进或回收

## 交接包

每章完成后写入本章交接包。下一轮应能仅凭交接包和核心台账继续工作。
