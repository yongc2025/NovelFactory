# 长篇项目协议

本协议用于管理长篇小说、连载小说、多卷本小说和大型改稿项目。

## 核心原则

长篇小说应被视为持续演化的项目状态，而不是一次性正文生成。

每一轮工作必须至少完成以下一种任务：

1. 建立项目状态
2. 读取项目状态
3. 生成当前阶段材料
4. 审核项目状态
5. 更新项目状态
6. 写入下一轮交接信息

## 标准工作顺序

```text
定位项目
→ 读取项目总览与最新状态
→ 判断当前阶段
→ 读取必要文件
→ 执行本轮任务
→ 检查连续性
→ 更新相关台账
→ 写入交接包
→ 更新最新状态
```

## 项目阶段

```text
立项
基础设定
世界观
角色设计
全书大纲
分卷大纲
章节表
章节卡
场景卡
正文草稿
审校
修订
导出
```

## 正文前置条件

进入正文草稿前，至少需要具备：

1. 项目圣经
2. 十二层创作台账
3. 角色手册
4. 当前卷目标
5. 当前章节卡
6. 当前场景卡
7. 人物认知状态

如果资料不完整，先恢复临时状态，并把不确定内容标记为“待确认”。

## 每轮必须输出

每次生成大纲、章节卡、场景卡、正文、审校或修订后，都要更新交接包。

交接包至少包含：

- 当前阶段
- 本轮完成内容
- 新增项目事实
- 人物状态变化
- 信息释放变化
- 伏笔变化
- 读者承诺变化
- 连续性风险
- 下一步建议
