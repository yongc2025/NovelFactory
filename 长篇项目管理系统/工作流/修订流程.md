# 修订流程

本流程用于处理润色、重写、结构调整、设定更新和追溯调整。

## 原则

不要把所有修改都当成文字润色。

先判断修改类型，再决定要更新哪些文件。

## 修改类型

### 1. 文风润色

通常影响：

- 正文草稿
- 审校报告

不改变项目事实。

### 2. 章节结构调整

通常影响：

- 章节卡
- 场景卡
- 正文草稿
- 章节表
- 交接包

适用于章节目标、场景顺序、结尾钩子或章节结果发生变化。

### 3. 设定更新

通常影响：

- 项目圣经
- 世界观手册
- 角色手册
- 连续性台账
- 后续大纲

适用于已经确认的项目事实发生变化。

### 4. 追溯调整

通常影响：

- 已写章节
- 信息释放表
- 人物认知状态
- 伏笔台账
- 读者承诺台账
- 修订记录

适用于过去章节中的事实、线索、发现时机或关系状态需要重新安排。

## 标准流程

```text
确认修改要求
→ 判断修改类型
→ 列出受影响文件
→ 分析下游影响
→ 生成修改方案
→ 执行修改
→ 更新相关台账
→ 写入修订记录
→ 更新最新状态与交接包
```

## 修改方案模板

```markdown
# 修改方案

## 修改编号

## 修改类型

## 用户要求

## 受影响文件

## 影响分析

## 执行步骤

## 台账更新

## 风险

## 回滚说明
```

## 审核问题

提交修改前检查：

- 人物是否提前知道不该知道的内容？
- 能力成长是否缺少过程？
- 伏笔是否失去后续作用？
- 读者承诺是否被提前或延后？
- 情绪线是否跳过必要阶段？
- 后续章节是否依赖旧版本？
