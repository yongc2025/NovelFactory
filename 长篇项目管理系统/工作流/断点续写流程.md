# 断点续写流程

本流程用于新会话、上下文缺失、长时间中断后继续创作。

## 目标

在不重新阅读全书的情况下，恢复足够可靠的项目状态，安全继续下一步创作。

## 优先读取文件

1. 项目清单
2. 最新状态
3. 当前章节表
4. 最近一章交接包
5. 人物认知状态
6. 伏笔台账
7. 读者承诺台账

## 恢复步骤

```text
确认项目编号
→ 确认当前阶段
→ 读取最新状态
→ 读取最近交接包
→ 恢复已确认事实
→ 标记待确认事实
→ 恢复主角认知状态
→ 恢复当前情绪阶段
→ 恢复未完成冲突
→ 确认下一步安全动作
```

## 不允许

- 不把缺失信息直接当成事实。
- 不在人物认知状态不明时直接写正文。
- 不在伏笔状态不明时回收重要线索。
- 不用大段设定解释填补空白。

## 断点恢复报告

```markdown
# 断点恢复报告

## 项目编号

## 恢复到的阶段

## 已确认事实

## 待确认内容

## 主角当前状态

## 当前冲突

## 未兑现承诺

## 连续性风险

## 下一步安全动作
```
