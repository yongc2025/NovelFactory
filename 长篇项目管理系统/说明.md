# 长篇项目管理系统

这是 NovelFactory 仓库中的独立模块，用来管理长篇小说从立项、设定、大纲、章节、正文、审校到修订的全过程记录。

## 定位

本系统不直接替代小说写作技能，而是作为文件化项目管理层。

```text
小说写作技能 = 创作与审校方法
长篇项目管理系统 = 项目状态文件体系
GitHub 仓库 = 长期保存与版本记录
```

## 核心原则

长篇小说不是一次性文本生成，而是持续演化的项目状态。

每一轮创作都应遵循：

```text
读取项目状态 -> 生成当前阶段内容 -> 审核连续性 -> 更新项目记录 -> 写入交接包
```

## 管理内容

- 项目圣经
- 十二层创作台账
- 世界观资料
- 角色手册
- 人物认知状态
- 情绪线
- 章节表
- 章节卡
- 场景卡
- 正文草稿
- 审校报告
- 伏笔台账
- 读者承诺台账
- 修订记录
- 断点续写交接包

## 建议项目目录

```text
projects/项目编号/
├── 00_项目总览/
├── 01_基础设定/
├── 02_世界观/
├── 03_角色/
├── 04_大纲/
├── 05_章节/
├── 06_台账/
└── 07_导出/
```

## 默认规则

在没有项目圣经、章节卡和场景卡之前，不直接进入正文写作。
