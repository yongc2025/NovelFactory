# 命名规范

本文件用于统一长篇项目管理系统中的目录名、文件名和模板字段。

## 总原则

1. 面向中文小说创作，目录名和文件名优先使用中文。
2. 文件名应表达用途，不使用含糊缩写。
3. 同一概念只使用一个固定名称。
4. 工作流文件描述“怎么做”。
5. 模板文件描述“记录什么”。
6. 台账文件描述“变化如何追踪”。

## 顶层目录

```text
长篇项目管理系统/
├── 说明.md
├── 系统设计.md
├── 方法论映射.md
├── 命名规范.md
├── 工作流/
├── 模板/
└── 示例项目/
```

## 项目实例目录

```text
projects/项目编号/
├── 00_项目总览/
├── 01_基础设定/

├── 03_角色/

```

## 统一术语

| 推荐术语 | 不推荐写法 | 说明 |
|---|---|---|
| 项目圣经 | bible、设定总表 | 全项目最高级别设定文件 |
| 十二层创作台账 | 12层表、方法论表 | 方法论状态记录 |
| 角色手册 | 人物表、角色表 | 角色设定与成长记录 |
| 世界观手册 | 世界设定、世界书 | 世界压力、规则、势力记录 |
| 章节表 | outline、章节大纲表 | 全书或分卷章节级结构 |
| 章节卡 | chapter card | 单章目标、阻碍、变化、钩子 |
| 场景卡 | scene card | 场景冲突单元 |
| 人物认知状态 | knowledge state | 人物当前知道、误解、未知内容 |
| 情绪线 | emotion arc | 人物心理变化记录 |
| 信息释放表 | information release | 设定与真相的分层揭示记录 |
| 伏笔台账 | foreshadowing ledger | 伏笔埋设、推进、回收记录 |
| 读者承诺台账 | reader promise ledger | 类型承诺与满足感兑现记录 |
| 连续性台账 | continuity ledger | 设定、时间、人物状态一致性记录 |
| 交接包 | handoff package | 每轮工作后的续写交接记录 |
| 修订记录 | revision log | 修改原因、影响范围、状态记录 |

## 文件命名规则

使用中文名和 `.md` 后缀：

```text
项目圣经.md
章节卡.md
交接包.md
人物认知状态.md
```

章节目录使用三位编号：

```text
第001章/
第002章/
第003章/
```

重大修订记录可以加日期：

```text
修订记录_2026-06-12.md
```

## Git 提交信息建议

```text
添加中文项目管理系统模板
补充方法论映射
更新章节生产流程
新增示例项目骨架
修订项目圣经模板
```

## 英文目录处理原则

历史遗留英文目录暂不删除。中文系统稳定后，再统一迁移、归档或删除，避免误删已有内容。
