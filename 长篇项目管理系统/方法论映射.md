# 方法论映射

本文件用于说明：长篇项目管理系统如何承接“小说创作通用方法论：从灵感到成稿的 12 层系统”。

## 核心定位

12 层方法论是创作标准层，回答“故事为什么成立、章节为什么有效、人物为什么值得读”。

长篇项目管理系统是执行与记录层，回答“这些判断如何变成文件、台账、流程和可持续更新的项目状态”。

```text
12 层方法论 = 写作判断标准
长篇项目管理系统 = 文件化执行系统
GitHub = 状态保存与版本管理
```

## 第一层：核心命题 → 项目圣经

方法论问题：

- 主角曾经被什么塑造？
- 他来到新故事后想要什么？
- 故事最终要让他明白什么？

项目文件落点：

- `模板/项目圣经.md`
- `模板/十二层创作台账.md`

必须记录：

- 表层类型
- 深层命题
- 主角被什么塑造
- 主角想要什么
- 故事迫使主角理解什么
- 主题句

## 第二层：读者承诺 → 读者承诺台账

方法论问题：

- 本书给读者什么稳定回报？
- 哪些满足感可以早给？
- 哪些满足感必须延迟？

项目文件落点：

- `项目圣经.md`
- `读者承诺台账.md`
- `章节卡.md`

必须记录：

- 立即承诺
- 重复奖励
- 延迟兑现
- 每章承诺推进情况
- 已兑现内容
- 未兑现内容

## 第三层：主角伤口 → 角色手册与情绪线

方法论问题：

- 主角为什么不是普通能人？
- 他为什么必须赢？
- 他的敏感点、恐惧、防御机制是什么？

项目文件落点：

- `项目圣经.md`
- `角色手册.md`
- `情绪线.md`

必须记录：

- 核心伤口
- 错误信念
- 防御机制
- 情绪触发点
- 每一阶段的心理变化

## 第四层：主角欲望 → 章节目标链

方法论问题：

- 主角现在想完成什么？
- 长期欲望如何拆成阶段性目标？
- 外部目标和内部需要是否一致或冲突？

项目文件落点：

- `项目圣经.md`
- `角色手册.md`
- `章节表.md`
- `章节卡.md`

必须记录：

- 外部欲望
- 内部欲望
- 阶段目标
- 当前章节目标
- 目标变化原因

## 第五层：世界压力 → 世界观手册

方法论问题：

- 世界如何压迫主角？
- 背景设定是否通过压力出现？
- 世界是否逼主角行动，而不是等待主角介绍？

项目文件落点：

- `世界观手册.md`
- `规则表.md`
- `章节表.md`

必须记录：

- 生存压力
- 阶层或制度压力
- 资源或技术压力
- 历史压力
- 这些压力在哪些章节显现

## 第六层：冲突系统 → 章节表与场景卡

- `章节卡.md`

- 历史冲突

## 工作流如何调用方法论

### 立项阶段

必须检查：

- 第一层：核心命题
- 第二层：读者承诺
- 第三层：主角伤口
- 第四层：主角欲望

### 世界观阶段

必须检查：

- 第五层：世界压力
- 第七层：信息释放

### 大纲阶段

必须检查：

- 第四层：主角欲望
- 第六层：冲突系统
- 第八层：人物弧光
- 第九层：章节设计

### 章节生产阶段

必须检查：

- 第七层：信息释放
- 第九层：章节设计
- 第十层：场景设计
- 第十一层：爽点设计

### 审校修订阶段

必须检查：

- 第十二层：改稿审计
- 第三层：主角伤口
- 第七层：信息释放
- 第八层：人物弧光

## 系统使用规则

1. 方法论负责判断内容是否成立。
2. 项目管理系统负责保存判断结果和后续变化。
3. 每个模板字段都应对应至少一个方法论层级。
4. 每次创作输出后，必须更新与本轮相关的项目文件。
5. 每次修改设定或章节结构后，必须更新对应台账和交接包。
6. 如果方法论判断与已有项目文件冲突，先生成修订方案，不直接覆盖原状态。

## 最终关系

```text
方法论管：为什么这样写
工作流管：先做什么后做什么
模板管：结果记录在哪里
台账管：变化如何持续追踪
Git 管：每次修改如何回滚和审查
```
