# CONTEXT — 流程回正

## 背景

当前仓库经过多轮前后端、文档、任务包改造后，已经出现流程与实现状态不一致的问题。根目录
`AGENTS.md` 要求每个实质性任务先创建或更新任务包，并在 `tasks/INDEX.md` 中锁定上下文；但当前工作树中
存在大量跨域改动，且部分“已完成”任务仍有未验证的契约问题。

## 已观察到的偏离

| 等级 | 偏离点 | 影响 |
|------|--------|------|
| P0 | 工作树存在大量跨前端、后端、文档的未收束改动 | 无法判断哪些变更属于哪个验收目标 |
| P0 | `0003-inline-editing` 标记完成，但前端审计发现 API 契约仍有阻塞问题 | “已完成”状态不可信，后续开发容易叠加错误 |
| P0 | `tasks/INDEX.md` 未体现当前流程回正任务 | Agent 无法按黄金路径锁定最新上下文 |
| P1 | `docs/TASKS.md` 有未入 `tasks/INDEX.md` 的伪任务 | 任务系统与文档系统分叉 |
| P1 | 部分任务包缺少 `CONTEXT/ACCEPTANCE/TODO` | 验收标准不完整 |
| P1 | 前端文件行数与 `any` 使用超过强制规范 | 维护风险和类型契约风险升高 |

## 本任务范围

- 创建完整任务包，作为当前流程回正入口。
- 产出项目流程偏离审计文档。
- 同步 `tasks/INDEX.md` 与 `docs/TASKS.md`。
- 将已发现的 P0/P1 问题收束成后续必须建包处理的 backlog。

## 非目标

- 不在本任务中修复前后端运行时代码。
- 不重写历史文档。
- 不清理或回滚已有用户改动。
- 不创建 Git commit。
