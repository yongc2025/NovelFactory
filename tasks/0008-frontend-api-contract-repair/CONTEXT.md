# CONTEXT — 前端契约修复

## 背景

`ProjectDetail.vue` 当前存在阶段确认卡展示位置错误、概览页误显示确认卡、重新生成缺少全周期加载状态、tab key 与后端 stage key 不一致等问题。该问题会影响选题、世界观、角色、大纲、元数据、正文、审校等核心流水线体验。

## 已确认问题

- 项目概览页显示 `StageConfirm`，用户在看不到阶段内容时可能误操作。
- 选题页底部全局确认卡与卡片内“采用此方案”重复，且布局不符合先审阅再操作的习惯。
- 世界观等长内容页面的确认卡在底部，用户需要滚动到底部才能重生成。
- `characters/chapters` 前端 tab key 与后端 `character/draft` stage key 不一致。
- `feedback` 未传给 API，编辑反馈不可用。
- 重新生成依赖轮询，但缺少全周期遮罩与防重复点击。

## 范围

- 修复详情页阶段动作与布局。
- 保持现有 API，不新增后端接口。
- 不做大范围视觉重设计。
