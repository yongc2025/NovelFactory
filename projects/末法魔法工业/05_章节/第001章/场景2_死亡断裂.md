# 第001章 场景2：死亡断裂

## 基本信息

- 项目编号：末法魔法工业
- 章节编号：第001章
- 场景编号：场景2
- 场景名称：死亡断裂
- 场景定位：前世死亡场景 / 署名伤口定型 / 穿越前断裂点
- POV：陆衡
- 时间：前世，场景1之后，同夜至凌晨
- 地点：实验室、设备间、实验楼走廊或返程途中
- 本场任务：让过劳、署名被夺、孤立无援和身体崩溃叠加，完成死亡断裂，但不解释穿越机制。

---

# 01 场景一句话

陆衡保存完最后一份原始记录后，身体在过劳和突发事故中崩溃；意识断裂前，他最后看见的不是死亡本身，而是报告里那个被挪走的名字。

---

# 02 场景核心目的

本场必须完成四件事：

1. 承接场景1：陆衡已经确认署名和贡献描述被改写。
2. 加深伤口：他不是输给某个坏人，而是输给一整套不承认劳动细节的权力结构。
3. 完成死亡：过劳、事故、孤立和不甘叠加，而非华丽牺牲。
4. 为场景3异界醒来做断片式转场。

本场不写异界设定，不解释魂穿原因。

---

# 03 场景冲突单元

| 项目 | 内容 |
|---|---|
| 陆衡想要 | 保存证据，保住自己对成果的最后证明 |
| 阻挡力量 | 身体极限、深夜孤立、设备风险、制度冷处理 |
| 陆衡可失去 | 生命、名字、成果归属、最后的公平信念 |
| 对方可失去 | 基本没有即时损失，这正是压迫感来源 |
| 冲突本质 | 个体记录 vs 结构性抹除；身体极限 vs 继续自证 |
| 场景胜负 | 陆衡保存了记录，但没能保住自己 |
| 胜负代价 | 死亡发生，署名伤口彻底定型 |

---

# 04 死亡原因建议

推荐采用：

```text
长期过劳 + 深夜实验楼设备/环境事故 + 身体崩溃
```

可选具体形式：

1. **设备间事故**：他去确认一组仍在运行的热处理 / 老化测试设备，因眩晕、低血糖、设备异常叠加出事。
2. **实验楼走廊坠倒**：保存记录后准备离开，耳鸣、胸闷、视野发黑，在楼梯或走廊摔倒。
3. **返程事故**：深夜骑车或走出实验楼时因极度疲劳发生交通事故。

最推荐：**设备间事故**。

理由：

- 与材料、热处理、设备、实验环境相关。
- 更能体现“工作场域吞掉了他”。
- 不需要外部戏剧化车辆事故。
- 可和后续灰炉、热源、设备故障形成隐性呼应。

---

# 05 场景情绪线

| 节点 | 表层情绪 | 深层情绪 | 外在表现 |
|---|---|---|---|
| 保存记录 | 冷静、机械 | 只剩最后防御 | 文件命名、备份、核对时间戳 |
| 身体预警 | 压抑、迟钝 | 不愿承认撑不住 | 手抖、耳鸣、胸闷、看字重影 |
| 继续行动 | 麻木 | 不能就这样停下 | 仍去确认设备或记录 |
| 事故发生 | 断裂 | 不甘与空白同时出现 | 视野黑、身体失控、设备声远去 |
| 意识消失 | 极静 | 名字伤口定型 | 最后看见署名栏或文件名 |

---

# 06 场景认知边界

## 陆衡本场知道

- 自己的名字已经不在该在的位置。
- 对方不会在提交前真正改回去。
- 自己至少要保住原始记录和版本证据。
- 身体已经不太对劲，但他低估了严重性。

## 陆衡本场不知道

- 自己会死亡。
- 自己会魂穿。
- 自己会醒在莱恩身体里。
- 自己的署名伤口会成为异界制度建构的核心动力。

## 读者本场应该知道

- 陆衡不是主动选择穿越，而是被前世制度和身体极限推到断点。
- 他的死亡与“名字被夺”情绪绑定。
- 后续他对署名、记录、契约、参与者名字的敏感有来源。

## 禁止提前知道

- 不解释魂穿机制。
- 不出现莱恩名字。
- 不出现异界画面。
- 不出现魔法、灰炉、寒潮信息。

---

# 07 场景动作设计

## 陆衡动作

- 把原始数据复制到备用硬盘、网盘、邮件草稿或本地加密文件。
- 给备份文件命名，文件名里带日期、批次、实验条件。
- 核对修改记录时手指发抖。
- 想喝水，却发现杯子已经空了。
- 抬头时短暂眩晕，扶住桌角。
- 仍然决定去设备间确认最后一组数据或关闭设备。
- 走路时肩膀蹭到门框，反应慢半拍。
- 事故发生时，他本能先护住记录本 / 硬盘 / 文件夹。

## 环境动作

- 设备低鸣从背景逐渐放大。
- 热处理炉、老化箱、风机或警示灯形成压迫感。
- 走廊灯闪烁，实验楼空无一人。
- 门禁声、风机声、设备报警声变得像从水下传来。
- 屏幕保护程序亮起，署名栏仍停在那一页。

## 外界反应

本场不需要立即写抢救过程。

如果需要，可用极少量碎片：

- 远处有人喊他的名字。
- 手机落在地上震动。
- 屏幕上消息继续跳出“先提交”。
- 没有人真正知道他刚刚保存了什么。

---

# 08 对白与声音碎片

本场对白应少。

可使用消息、回忆、设备声替代对话。

## 可出现的消息碎片

- “先提交，别耽误节点。”
- “署名后面再协调。”
- “你太累了，早点休息。”
- “陆衡，别把事情弄复杂。”
- “项目是团队成果。”

## 陆衡可说的话

- “保存完就走。”
- “还有一组数据。”
- “版本记录不能丢。”
- “名字……不是形式。”

最后一句不宜喊出来，更适合低声或内心闪念。

---

# 09 场景节拍

## 节拍1：保存证据

承接场景1。

陆衡没有再争辩，而是回到最熟悉的防御方式：记录。

### 重点

- 备份原始数据。
- 保存报告旧版本。
- 截图或导出修改记录。
- 写下实验批次和条件。

### 情绪

冷，不哭，不喊。

---

## 节拍2：身体报警

身体开始出问题。

### 可写细节

- 心口闷。
- 眼前字重影。
- 耳鸣。
- 指尖麻。
- 胃里空得发痛。
- 手指敲错文件名。

### 注意

陆衡会把这些归因于熬夜、低血糖、咖啡喝多了，而不是“我要死了”。

---

## 节拍3：仍然去确认设备

他没有回宿舍或医院，而是去确认最后一组实验设备。

### 逻辑

这不是他蠢，而是长期科研训练后的惯性：

```text
设备还在跑。
数据还没落盘。
失败批次还没记录。
我不能现在停。
```

---

## 节拍4：事故发生

推荐写法：冷硬、突然。

不要写慢镜头英雄式死亡。

### 示例方向

- 他伸手去关掉异常报警。
- 眩晕让他判断慢了半拍。
- 脚下一空或身体撞上设备边缘。
- 高温、金属、警报、黑暗同时压下来。

### 注意

事故不必写得血腥。关键是意识断裂。

---

## 节拍5：最后记忆

最后记忆不应是疼痛，而应是名字。

可选落点：

1. 屏幕上报告署名栏。
2. 备份文件名里的“陆衡”。
3. 消息里“团队成果”。
4. 他自己写下的实验记录签名。

推荐落点：

```text
备份文件名里的“陆衡”。
```

理由：

- 与后续“莱恩”形成强对照。
- 不是别人写的名字，而是他最后自己留下的名字。

# 10 细节母题

| 母题 | 功能 |
|---|---|
| 备份文件名 | 陆衡最后保护自己的方式 |
| 时间戳 | 事实存在过的证据 |
| 空杯子 | 过劳和身体透支 |
| 设备低鸣 | 工作场域吞噬感 |
| 报警灯 | 事故前兆 |
| “先提交”消息 | 权力结构继续运转 |
| “陆衡”二字 | 前世名字最后一次被主动保留 |

---

# 11 本场禁区

1. 不写陆衡死前长篇控诉。
2. 不写华丽穿越光效。
3. 不写神秘声音召唤。
4. 不写他主动选择重生。
5. 不写上级直接杀人或阴谋谋杀。
6. 不让事故像爽文复仇前奏。
7. 不让死亡轻飘，要有身体透支和制度冷感。
8. 不在本场出现“莱恩”。

---

# 12 本场输出要求

正文建议字数：800—1400字。

节奏建议：

```text
保存记录 → 身体报警 → 继续确认设备 → 事故突发 → 名字定格 → 黑暗
```

语言质感：

- 冷。
- 硬。
- 少抒情。
- 多动作。
- 让读者自己感到不甘。

---

# 13 场景结尾建议

推荐结尾方向：

```text
文件传输完成的提示框弹出。

备份文件名最后一段，是他的名字。

陆衡。

下一秒，设备低鸣被拉成一条很长的线。
他想伸手去按停止键，却先失去了脚下的地面。

屏幕还亮着。
他的名字停在那里。
```

然后进入场景3：异界醒来。

---

# 14 场景交接

进入场景3时必须承接：

- 陆衡前世死亡已完成。
- 他最后主动保留的是自己的名字。
- 意识断裂后，不直接解释穿越机制。
- 场景3第一感官应从“设备低鸣”切到“寒风 / 药味 / 壁炉灰 / 旧木床”。
- 场景3的第一个人物压力应来自巴伦称呼“莱恩少爷”。
