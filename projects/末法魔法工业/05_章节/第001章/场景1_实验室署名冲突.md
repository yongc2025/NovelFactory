# 第001章 场景1：实验室署名冲突

## 基本信息

- 项目编号：末法魔法工业
- 章节编号：第001章
- 场景编号：场景1
- 场景名称：实验室署名冲突
- 场景定位：开篇伤口场景 / 前世署名创伤定型
- POV：陆衡
- 时间：前世，深夜至凌晨
- 地点：材料实验室 / 办公室 / 实验楼走廊
- 本场任务：让读者看见陆衡的核心伤口不是“穷”或“穿越”，而是劳动成果和名字被制度性抹掉。

---

# 01 场景一句话

陆衡在过劳后的深夜确认实验结果，却发现关键报告中的署名和贡献描述被悄悄改写；他试图用记录和事实保护自己，但权力结构已经把他的名字从成果中心挪开。

---

# 02 场景核心目的

本场必须完成四件事：

1. 建立陆衡前世身份：材料学与机械制造交叉方向科研人。
2. 建立陆衡方法论：记录、复核、版本、测试条件。
3. 建立核心伤口：成果被夺、署名被挪走。
4. 建立性格底色：冷静、克制、压抑、不善表达情绪，但对记录和署名极端敏感。

本场不要解释穿越，不要提异界，不要写爽文式打脸。

---

# 03 场景冲突单元

| 项目 | 内容 |
|---|---|
| 陆衡想要 | 确认自己的实验结果、贡献和署名被如实保留 |
| 阻挡力量 | 导师 / 上级 / 项目组署名规则 / 即将提交的报告版本 |
| 陆衡可失去 | 成果归属、第一作者或核心贡献位置、最后的公平信念 |
| 对方可失去 | 项目署名分配权、资源整合权、对成果包装的控制 |
| 冲突本质 | 事实记录 vs 权力分配；劳动成果 vs 项目包装；署名真实 vs 组织话术 |
| 场景胜负 | 陆衡没有赢。他只保住了原始记录，但无法阻止报告被提交。 |
| 胜负代价 | 陆衡的错误信念崩塌：成果够硬并不会自动带来公平。 |

---

# 04 场景情绪线

| 节点 | 陆衡表层情绪 | 陆衡深层情绪 | 外在表现 |
|---|---|---|---|
| 进入场景 | 疲惫、麻木 | 仍相信数据能说话 | 低头核对数据，不抱怨 |
| 发现异常 | 冷静变硬 | 不敢相信名字被挪走 | 盯住署名栏，手指找笔 |
| 追问对方 | 克制 | 愤怒被理性压住 | 要求版本记录、原始记录、贡献说明 |
| 遭到敷衍 | 沉默 | 信任坍塌 | 句子变短，声音变低 |
| 场景结束 | 压住失控 | 伤口定型 | 保存记录，身体出现崩溃前兆 |

---

# 05 场景认知边界

## 陆衡本场知道

- 实验数据是真的。
- 关键工艺路线、失效分析或材料方案主要由自己完成。
- 报告版本已经被改动。
- 署名和贡献描述不再如实反映劳动。
- 对方想用“团队贡献”“项目安排”“先提交”压过事实。

## 陆衡本场不知道

- 自己很快会死亡。
- 自己会魂穿莱恩。
- “名字被夺”会在异界变成“另一个名字压身”。

## 读者本场应该知道

- 陆衡是专业、疲惫、被制度压迫的人。
- 他不是热血少年，也不是轻松开挂者。
- 他最不能忍受的是成果归属被模糊。

## 禁止提前知道

- 不提灰石领。
- 不提莱恩。
- 不提魔法。
- 不提寒潮。
- 不提后续工业路线。

---

# 06 场景人物动作设计

## 陆衡动作

- 盯着屏幕上的署名栏，不立刻说话。
- 指尖去摸桌边的笔，摸空后才发现笔在文件下面。
- 重新打开原始数据文件，核对时间戳。
- 把报告版本号、修改时间、贡献描述逐项记下来。
- 说话越来越短。
- 对方说“大家都有贡献”时，他不反驳“我最重要”，而是问：“原始记录谁做的？”
- 保存备份时手指发抖，但他先以为是低血糖或熬夜。

## 导师 / 上级动作

- 不需要正面恶毒。
- 可以只通过电话、消息、语音、会议纪要出现。
- 语气温和但压迫：
  - “先提交。”
  - “署名后面还能调整。”
  - “项目是团队成果。”
  - “你还年轻，以后机会很多。”
- 重点不是骂陆衡，而是把不公平说成正常流程。

## 环境动作

- 实验室深夜只剩设备低鸣。
- 冷掉的咖啡、泡开的泡面、反复修改的报告版本。
- 屏幕蓝光照在陆衡脸上。
- 手机消息不断跳，但没有一条真正解决问题。
- 走廊灯有一段坏掉，形成压抑气氛。

---

# 07 对白样本

## 陆衡

- “这一版是谁改的？”
- “原始记录在我这里。测试条件、失败批次、修改原因，都在。”
- “贡献说明为什么删掉了热处理段？”
- “署名顺序什么时候变的？”
- “先提交是什么意思？”
- “如果后面还能调整，现在为什么不能写清楚？”
- “这不是措辞问题。名字被拿掉了。”

## 上级 / 导师 / 项目负责人

- “陆衡，你先冷静。”
- “项目不是一个人的。”
- “你做了很多，大家都看得到。”
- “署名只是形式，成果能发出去才重要。”
- “年轻人不要在这种事上太较真。”
- “先提交，后面组里会协调。”
- “你要相信团队。”

## 陆衡反击边界

陆衡不能热血怒骂。更适合这样：

```text
他没有抬高声音，只把报告往回滚了三页。
“那就把版本记录打开。”
```

或者：

```text
“如果只是形式，”陆衡说，“为什么我的名字不能留在原来的位置？”
```

# 08 场景节拍

## 节拍1：过劳深夜

- 陆衡还在实验室。
- 他正在核对数据和报告。
- 先写疲惫，不先写冲突。
- 读者看见他已经习惯这种工作强度。

### 重点

不要用旁白说“他很累”，用动作写：

- 指尖敲错键。
- 咖啡冷掉。
- 他忘了上一顿饭什么时候吃。
- 他看数据时仍然准确。

---

## 节拍2：发现署名异常

- 他打开最新报告版本。
- 发现署名顺序、贡献说明、关键段落被改。
- 他第一反应不是爆发，而是复核。

### 重点

让读者看到他的职业反应：

```text
不是“他们害我”，而是“版本、时间、修改人、原始记录”。
```

---

## 节拍3：追问

- 他联系负责人。
- 对方用温和话术压住他。
- 冲突不靠吵，而靠话术和事实之间的错位。

### 重点

上级越温和，压迫越强。

---

## 节拍4：信念崩塌

- 陆衡意识到这不是误会。
- 他的记录是真的，但权力结构不承认。
- “成果够硬就会公平”的信念崩塌。

### 重点

不要写长篇心理独白。用一个动作定型：

```text
他把笔帽扣回去，又拔开。
最后在自己的备份文件名后面，加了一个日期。
```

---

## 节拍5：身体崩溃前兆

- 手抖、眩晕、胸闷、耳鸣。
- 他还在保存记录。
- 场景结束前，把情绪交给身体。

### 重点

为场景2死亡做铺垫。

---

# 09 细节母题

本场可以反复出现以下母题：

| 母题 | 功能 |
|---|---|
| 署名栏 | 核心伤口 |
| 版本记录 | 陆衡防御方式 |
| 冷咖啡 | 过劳状态 |
| 空白贡献说明 | 成果被抹掉 |
| 找笔动作 | 后续异界记录习惯的前身 |
| 屏幕蓝光 | 冷硬现实 |
| “先提交” | 权力话术 |

---

# 10 本场禁区

1. 不写上级露骨抢功大笑。
2. 不写陆衡当场打脸成功。
3. 不写陆衡长篇控诉科研制度。
4. 不写热血宣言。
5. 不解释死亡和穿越。
6. 不让读者觉得陆衡只是“社畜惨”，重点必须落在署名和成果归属。
7. 不让冲突变成普通职场吵架，要保持“记录事实无力对抗权力结构”的冷感。

---

# 11 本场输出要求

正文写作时建议字数：1200—1800字。

节奏建议：

```text
安静疲惫 → 发现异常 → 冷静复核 → 温和压迫 → 信念崩塌 → 身体崩溃前兆
```

本场结尾建议不要直接死亡，留给场景2：

```text
陆衡保存完最后一个备份。
他想站起来，却发现实验室的设备低鸣突然变得很远。
屏幕上，最新版本报告的署名栏仍亮着。
他的名字不在该在的位置。
```

---

# 12 场景交接

进入场景2时必须承接：

- 陆衡已发现署名被挪走。
- 他没有解决问题。
- 他保存了记录。
- 身体崩溃前兆已经出现。
- 他的核心伤口已经定型，但死亡尚未完成。
