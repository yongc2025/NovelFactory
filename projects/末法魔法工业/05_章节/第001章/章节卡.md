# 01 本章核心功能

本章不是完整解释世界观，也不是正式进入灰炉制造。

本章只完成四件事：

1. 写出陆衡前世伤口：成果被夺署名。
2. 写出陆衡死亡前的精神状态：过劳、压抑、冷静到麻木。
3. 写出“名字消失”这个核心痛点。
4. 章末让他在异界醒来，被巴伦称为“莱恩少爷”。

本章核心钩子：

```text
一个刚刚失去自己名字的人，醒来后被迫继承另一个欠债的名字。
```

---

# 02 本章四问

## 2.1 主角现在知道什么？

- 陆衡知道自己前世正在经历严重过劳。
- 陆衡知道自己的项目成果即将被上级和团队抢署名。
- 陆衡知道“成果够硬就能得到公平回报”这个信念正在崩塌。
- 章末醒来后，他只知道自己没在原来的实验室 / 医院，而是在陌生房间里。
- 他听见有人叫他“莱恩少爷”，但不知道莱恩是谁。

## 2.2 主角现在为什么行动？

- 前半章：他想确认署名、确认实验结果、确认自己的劳动没有被抹掉。
- 死亡前：他想阻止成果被空白化，但身体和权力结构都不允许。
- 醒来后：他本能求生，必须先判断环境，不能立刻暴露自己不是“莱恩”。

## 2.3 这个世界信息是怎么被主角知道的？

本章异界信息只允许通过以下方式出现：

- 陌生房间环境。
- 不属于自己的身体。
- 巴伦的称呼。
- 零碎感官：冷、旧木、壁炉灰、药味、窗外风声。
- 不能用旁白解释末法时代、灰石领、魔法体系。

## 2.4 本章结束变化是什么？

- 陆衡从“前世署名被夺者”变成“异界身份错位者”。
- 他的核心伤口从“我的名字被拿走”转化为“别人把另一个名字压到我身上”。
- 他还没有接受莱恩身份，只进入第一阶段：伪装求生。

# 03 章节结构

## 3.1 章节目标

让读者在第一章内记住三个东西：

1. 陆衡最痛的是署名和成果归属。
2. 陆衡不是轻松穿越，而是在极度疲惫与不公中死亡。
3. “莱恩·格雷”这个名字不是爽点，而是新的压力。

## 3.2 阻碍

- 前世上级掌握项目、署名、发表和资源分配权。
- 陆衡身体过劳，精神濒临崩断。
- 他习惯用记录和数据保护自己，但制度不承认他的防御。
- 醒来后，他无法确认环境，无法解释自己是谁。
- 巴伦的称呼和凝视形成第一重身份压力。

## 3.3 行动

- 陆衡检查实验数据和报告署名。
- 他发现自己的名字被边缘化或从关键位置移除。
- 他试图追问或确认，但得到模糊、敷衍、权威式回应。
- 身体崩溃或事故发生。
- 醒来后，他不喊、不乱动，先观察。
- 听见“莱恩少爷”后，他压住反应，选择不暴露。

## 3.4 变化

- 前世：陆衡确认“成果够硬就会公平”是错的。
- 死亡瞬间：署名伤口定型。
- 异界醒来：身份排斥线启动。
- 章末：他第一次被迫面对“莱恩”这个名字。

## 3.5 章末钩子

建议章末停在以下信息组合：

```text
陌生老仆站在床边，称他为“莱恩少爷”。
陆衡没有立刻回应。
因为这个名字不是他的。
但门外的寒风、老人的眼神、房间里的旧徽章，都在逼他先活下去。
```

不直接解释魂穿，不解释灰石领全貌。

# 04 场景列表

## 场景一：实验室 / 报告署名

- 地点：前世实验室或办公室
- 时间：深夜 / 清晨前
- POV：陆衡
- 登场角色：陆衡、导师 / 上级、同组人员可只用电话、消息或背影处理
- 场景功能：建立署名伤口

### 场景冲突

| 项目 | 内容 |
|---|---|
| 陆衡想要 | 确认自己的成果和署名不被抹掉 |
| 阻碍者想要 | 快速提交成果，弱化陆衡贡献，保持权力分配 |
| 双方筹码 | 陆衡有实验记录和实际贡献；对方有职称、项目权、发表权 |
| 场景压力 | 陆衡身体过劳，时间紧，制度不在他这边 |
| 转折点 | 他发现署名位置变化或贡献表述被改写 |
| 场景结果 | 陆衡无法阻止，只能把愤怒压进记录动作里 |

### 重点动作

- 陆衡盯着署名栏。
- 手指去找笔。
- 他没有先骂人，而是要求“原始记录”和“版本记录”。
- 对方用“先提交”“后面再说”“大家都有贡献”敷衍。

### 避免内容

- 不写长篇科研职场说明书。
- 不把上级写成夸张反派。
- 不让陆衡发表热血宣言。

---

## 场景二：死亡 / 断裂

- 地点：实验室、回程路上或设备间
- 时间：同夜
- POV：陆衡
- 登场角色：陆衡
- 场景功能：让过劳、事故、署名伤口叠加，完成死亡转场

### 场景冲突

| 项目 | 内容 |
|---|---|
| 陆衡想要 | 继续撑住，保住证据和名字 |
| 阻碍 | 身体极限、事故、孤立无援 |
| 场景压力 | 疲惫、低血糖、眩晕、设备警报或交通风险 |
| 转折点 | 视线落到“名字缺席”的文件 / 屏幕 / 消息 |
| 场景结果 | 意识断裂，死亡发生 |

### 重点动作

- 他试图保存或复制记录。
- 指尖发抖。
- 屏幕上的署名栏、文件名或消息成为最后记忆。
- 不能写成华丽死亡，要冷、硬、突然。

---

## 场景三：异界醒来

- 地点：灰石领城堡卧室
- 时间：冬季前夕，寒潮将至
- POV：陆衡
- 登场角色：陆衡、巴伦
- 场景功能：启动身份错位与伪装求生

### 场景冲突

| 项目 | 内容 |
|---|---|
| 陆衡想要 | 判断环境，确认自己是否还活着，避免暴露异常 |
| 巴伦想要 | 确认少爷是否清醒，观察病后变化 |
| 双方筹码 | 陆衡有观察能力但无本地知识；巴伦有本地知识和身份判断权 |
| 场景压力 | 陌生身体、陌生称呼、巴伦凝视、门外寒风 |
| 转折点 | 巴伦称呼“莱恩少爷” |
| 场景结果 | 陆衡没有承认身份，只选择先伪装 |

### 重点动作

- 陆衡醒来后不立刻说话。
- 先观察手、房间、窗、炉灰、衣物。
- 对“少爷”反应慢半拍。
- 听到“莱恩”时，内心排斥，但外在压住。

### 避免内容

- 不让主角立刻接受穿越。
- 不让主角直接喊“我不是莱恩”。
- 不让巴伦立刻信任或解释一切。
- 不解释完整灰石领设定。

# 05 角色 / 情绪 / 认知联动检查

## 5.1 陆衡 / 莱恩

### 本章认知

- 知道前世成果被夺署名。
- 醒来后知道自己处于陌生身体和陌生环境。
- 不知道莱恩是谁。
- 不知道灰石领、末法时代、魔法体系。

### 本章情绪

- 前半：疲惫、压抑、愤怒被理性包住。
- 死亡前：不甘、冷硬、崩断。
- 醒来后：警惕、错位、排斥、求生。

### 本章防御方式

- 前世：用记录、数据、版本、署名来防御。
- 异界：用沉默、观察、少说话来防御。

### 本章不可写错

- 不能让他热血反抗成功。
- 不能让他穿越后立刻理清世界。
- 不能让他自然接受“莱恩”。

---

## 5.2 巴伦

### 本章认知

- 知道少爷病后醒来。
- 知道少爷可能性情变化。
- 不知道魂穿。
- 不知道陆衡前世。

### 本章情绪

- 克制、担忧、怀疑、试探。

### 本章防御方式

- 礼法称呼。
- 观察少爷反应。
- 用熟悉称谓测试身份连续性。

### 本章不可写错

- 不能立刻信任主角。
- 不能解释太多设定。
- 不能表现得像工具人老管家。

---

## 5.3 原莱恩

### 本章呈现方式

原莱恩不直接出场，只通过以下方式存在：

- 陌生身体。
- 房间旧物。
- 格雷家徽章。
- 巴伦称呼。
- 可能的未签文件或药味。

### 本章作用

让陆衡意识到：

```text
这个名字不是空的。
有人认识莱恩。
有人期待莱恩醒来。
但我不是他。
```

# 06 信息释放控制

## 本章可以释放

- 陆衡前世科研身份。
- 陆衡成果被夺署名。
- 陆衡过劳死亡。
- 陆衡醒在陌生身体。
- 巴伦称他为莱恩少爷。
- 房间属于一个破落贵族家庭。
- 外面很冷，寒潮氛围可用感官暗示。

## 本章禁止释放

- 完整魔法体系。
- 末法时代历史。
- 灰炉路线。
- 灰石领完整债务和税务结构。
- 行会、商会、军需署、王室财政派。
- 烬日战争、地脉锚点、调律塔。
- 原莱恩完整人生。

# 07 爽点与压迫点

## 本章不是常规爽点章

本章爽点不在打脸，而在建立强钩子：

```text
署名被夺 → 死亡 → 醒来后被另一个名字压住。
```

## 本章压迫点

- 劳动成果被夺。
- 权力结构无声碾压。
- 身体过劳崩溃。
- 身份错位。
- 巴伦的称呼不是救命，而是压力。

## 本章微爽点

- 陆衡临死前仍试图保留记录。
- 醒来后没有慌乱，而是先观察。
- 他没有接受“莱恩”，但选择先活下去。
