# 输出格式

本文件定义长篇小说项目总管的默认输出结构。

## 默认结构

除非用户只要简短回答，否则使用：

```markdown
## 任务判断

说明用户请求属于哪个阶段。

## 读取/依赖的项目状态

列出本轮需要读取或依赖的文件。

## 方法论检查

说明触发 12 层方法论中的哪些层。

## 正式产出

输出本轮主要成果。

## 应更新文件

列出建议写入或更新的项目文件。

## 交接包

记录下一轮继续工作所需信息。

## 下一步建议

给出一个最合理的下一步。
```

## 创建项目时

输出：

```markdown
## 项目初始化判断

## 建议项目编号

## 需要创建的目录

## 需要创建的文件

## 初始项目圣经

## 初始十二层创作台账

## 下一步
```

## 章节规划时

输出：

```markdown
## 章节位置

## 四问门槛

## 章节五要素

## 场景列表

## 方法论风险

## 应更新文件
```

## 审校时

输出：

```markdown
## 审校对象

## 六线审计

## 12 层方法论问题

## 修改建议

## 必须更新的台账
```

## 修订时

输出：

```markdown
## 修订类型

## 原状态

## 新状态

## 影响范围

## 修改方案

## 应更新文件

## 回滚说明
```

## GitHub 写入后

输出：

```markdown
## 已写入

- 文件路径
- 分支
- 提交说明

## 未写入但建议后续处理

## 下一步
```

## 语言要求

- 默认中文。
- 文件名、目录名、模板字段优先中文。
- `SKILL.md`、`agents/openai.yaml` 等 Skill 规范文件名保留英文。
