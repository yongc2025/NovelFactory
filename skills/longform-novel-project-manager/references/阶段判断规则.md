# 阶段判断规则

本文件用于判断用户请求属于哪个长篇项目阶段。

## 阶段列表

```text
立项
基础设定
世界观

全书大纲
分卷大纲

场景卡

```

## 判断规则

### 立项

用户要求：

- 想写一本新小说
- 只有灵感
- 需要确定题材、卖点、主角、读者承诺

应输出：

- 项目清单草案
- 项目圣经初版
- 十二层创作台账初版

### 基础设定

用户要求：

- 完善核心命题
- 梳理主角伤口和欲望
- 确定读者承诺

应读取：

- 项目圣经
- 十二层创作台账

### 世界观

用户要求：

- 设计世界规则
- 设计势力
- 设计资源、技术、制度、历史压力

应读取：

- 项目圣经
- 世界观手册
- 方法论映射

### 角色设计

用户要求：

- 设计主角
- 设计配角
- 梳理人物弧光
- 设计关系变化

应读取：

- 项目圣经
- 角色手册
- 情绪线

### 大纲

用户要求：

- 写全书大纲
- 写分卷大纲
- 排章节
- 设计前 10 章

应读取：

- 项目圣经
- 十二层创作台账
- 章节表
- 角色手册
- 世界观手册

### 章节卡

用户要求：

- 设计某一章
- 继续下一章但还没写正文
- 先规划章节

应输出：

- 章节卡
- 必要时补场景卡

### 场景卡

用户要求：

- 拆场景
- 设计冲突单元
- 规划正文段落前的场景结构

应输出：

- 场景卡
- 场景冲突检查

### 正文草稿

用户要求：

- 写正文
- 扩写某章
- 根据章节卡写一章

前置要求：

- 必须有章节卡
- 最好有场景卡
- 必须明确人物认知状态

### 审校

用户要求：

- 检查这一章
- 看看哪里崩了
- 按 12 层方法论审稿

应输出：

- 审校报告
- 应更新文件清单

### 修订

用户要求：

- 改设定
- 改章节结构
- 延后或提前某个信息
- 重写某章

应输出：

- 修订方案
- 影响范围
- 修订记录

### 断点续写

用户要求：

- 继续上次
- 从项目状态恢复
- 不记得写到哪里

应读取：

- 最新状态
- 交接包
- 章节表
- 人物认知状态

应输出：

- 断点恢复报告
- 下一步安全动作
