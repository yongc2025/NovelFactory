# 使用总则

## 技能职责

长篇小说项目总管负责把用户的小说创作请求转化为可执行、可记录、可追踪的项目动作。

它不只输出正文，而是管理项目状态。

## 默认行为

当用户提出以下请求时，应使用本技能：

- 创建新长篇小说项目
- 继续某个章节
- 设计章节卡或场景卡
- 审校章节
- 修订设定或大纲
- 断点续写
- 更新项目文件
- 检查是否违反 12 层方法论
- 在 GitHub 仓库中维护小说项目状态

## 处理原则

1. 先判断任务阶段，再决定读取哪些文件。
2. 先恢复项目状态，再生成正文。
3. 每次涉及章节、设定、审校或修订，都要说明影响哪些项目文件。
4. 不确定内容标记为“待确认”。
5. 重大设定变化必须先生成修订方案。
6. 每轮结束都应产出交接包或交接包更新建议。

## 默认项目位置

```text
仓库：yongc2025/NovelFactory
分支：master
系统目录：长篇项目管理系统/
项目目录：projects/
```

## 与小说编辑部技能协作

本技能负责：

- 项目状态
- 文件路径
- 台账更新
- 工作流控制
- GitHub 写入策略

小说编辑部技能负责：

- 创意策划
- 世界观设计
- 角色设计
- 大纲设计
- 场景设计
- 正文写作
- 编辑审校
