# GitHub 项目读写协议

## 默认位置

```text
仓库：yongc2025/NovelFactory
分支：master
系统目录：长篇项目管理系统/
项目目录：projects/
```

## 写入原则

1. 默认只写 `master` 分支。
2. 除非用户明确要求，不写 `main` 分支。
3. 新项目文件应放入 `projects/项目编号/`。
4. 系统规则文件应放入 `长篇项目管理系统/`。
5. Skill 文件应放入 `skills/longform-novel-project-manager/`。
6. 写入前先检查目标文件是否存在。
7. 已存在文件需要先读取当前内容和 SHA，再更新。
8. 不确定是否覆盖时，先生成修改方案。

## 项目文件更新顺序

创建或继续项目时，优先维护：

1. `00_项目总览/项目清单.md`
2. `00_项目总览/最新状态.md`
3. `01_基础设定/项目圣经.md`
4. `01_基础设定/十二层创作台账.md`
5. `04_大纲/章节表.md`
6. `05_章节/第XXX章/交接包.md`

## 每轮结束必须说明

- 写入了哪些文件
- 哪些文件需要后续更新
- 是否涉及设定变化
- 是否涉及修订记录
- 下一轮应该先读取哪些文件

## 禁止行为

- 不在未确认分支时写入。
- 不把项目文件写到系统模板目录。
- 不把模板当成真实项目状态。
- 不直接删除历史目录。
- 不在没有说明影响范围时覆盖已有项目文件。
