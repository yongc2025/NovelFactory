# NovelFactory 产品需求规格书 (PRD)

> 版本：1.0 | 更新日期：2026-06-12
> 本文档定义 NovelFactory 系统的产品需求，是开发和验收的权威依据。

---

## 1. 产品定义

### 1.1 产品定位

**NovelFactory** 是一个人的自动化小说工厂。

从一个灵感/选题出发，通过 10 个 AI 角色协作，自动生成完整小说，并适配多平台分发形态。

### 1.2 目标用户

| 用户类型 | 特征 | 核心需求 |
|----------|------|----------|
| 个人创作者 | 有写作想法，但缺乏时间或技巧 | 快速将灵感转化为完整小说 |
| 自媒体从业者 | 需要持续产出内容 | 高效生产可变现的内容 |
| 网文作者 | 需要辅助工具 | 提升写作效率，保持质量 |

### 1.3 核心价值

**每天只需 1-2 小时进行审核决策，AI 即可生产可变现内容。**

关键指标：
- 从灵感到完整小说：≤ 3 天
- 单章生成时间：≤ 30 秒
- 角色一致性：100%（无名字漂移）
- 伏笔回收率：≥ 90%

## 2. 用户角色与场景

### 2.1 主要用户角色

**创作者（Creator）**
- 身份：个人创作者或自媒体从业者
- 目标：快速生产高质量小说内容
- 痛点：缺乏时间、写作技巧不足、长篇一致性难保证

### 2.2 核心使用场景

**场景 1：从零开始创作**
```
用户输入一个灵感（如"重生复仇打脸"）
→ AI 生成多个选题方案
→ 用户选择方案
→ AI 自动生成世界观、角色、大纲
→ AI 批量生成正文
→ 用户审核并导出
```

**场景 2：续写已有作品**
```
用户导入已有大纲
→ AI 基于大纲生成场景细纲
→ AI 生成正文
→ 用户审核修改
→ 导出发布
```

**场景 3：多平台分发**
```
用户完成小说创作
→ 选择目标平台（番茄/小红书/短剧）
→ AI 生成对应格式
→ 导出适配内容
```

## 3. 功能需求

### 3.1 核心功能（P0）

#### F01: 项目创建

| 属性 | 描述 |
|------|------|
| 功能ID | F01 |
| 功能名称 | 项目创建 |
| 优先级 | P0 |
| 描述 | 通过向导式界面创建小说项目，支持 5 级参数配置 |

**输入参数：**
- Level 1（必填）：灵感/前提、目标平台、篇幅类型
- Level 2（选填）：题材定位、目标读者、内容基调
- Level 3（选填）：角色预设（主角/反派/配角）
- Level 4（选填）：世界观约束
- Level 5（高级）：生成策略（字数/章数/爽点密度等）

**验收标准：**
- [ ] 支持 26 项参数配置
- [ ] 根据篇幅类型自动推算默认值
- [ ] 快速模式（4步）和完整模式（8步）可选
- [ ] 项目创建后自动进入流水线

#### F02: 叙事引擎（8 阶段流水线）

| 属性 | 描述 |
|------|------|
| 功能ID | F02 |
| 功能名称 | 叙事引擎 |
| 优先级 | P0 |
| 描述 | 8 阶段自动流水线，从选题到元数据生成 |

**阶段定义：**

| 阶段 | 名称 | 输入 | 输出 | 负责角色 |
|------|------|------|------|----------|
| 1 | 选题评估 | 灵感/前提 | 3-5 个选题方案 | 选题策划师 |
| 2 | 世界观搭建 | 选中方案 | 世界观设定 | 世界观设计师 |
| 3 | 角色设计 | 世界观 | 角色卡（12维） | 角色设计师 |
| 4 | 大纲编剧 | 角色+世界观 | 章节大纲 | 大纲编剧 |
| 5 | 元数据生成 | 大纲 | 书名/简介/标签 | 元数据生成器 |
| 6 | 场景细纲 | 大纲 | 场景细纲 | 场景编剧 |
| 7 | 正文生成 | 场景细纲 | 章节正文 | 正文作者 |
| 8 | 编辑审校 | 正文 | 审校报告 | 编辑审校 |

**验收标准：**
- [ ] 支持完整流水线一键执行
- [ ] 支持单阶段独立执行
- [ ] 支持阶段间人工审核确认
- [ ] 支持断点续作（刷新页面后恢复进度）

## 5. 数据模型

### 5.1 核心实体

```
Project (项目)
├── WorldSetting (世界观)
├── Character (角色)
│   └── GrowthState (成长状态)
├── Outline (大纲)
│   └── Chapter (章节)
│       └── Scene (场景)
├── Draft (正文)
├── Review (审校)
├── Foreshadow (伏笔)
└── Metadata (元数据)
```

### 5.2 关键关系

- 一个 Project 有多个 Character
- 一个 Project 有一个 Outline
- 一个 Outline 有多个 Chapter
- 一个 Chapter 有多个 Scene
- 一个 Chapter 有一个 Draft
- 一个 Chapter 有一个 Review
- 一个 Project 有多个 Foreshadow

## 7. 用户流程

### 7.1 主流程

```
┌─────────────┐
│   仪表盘    │ ← 项目列表、统计数据
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  创建项目   │ ← 5级参数向导
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 项目详情页  │ ← 流水线控制、进度展示
└──────┬──────┘
       │
       ├─→ 选题审核 → 世界观审核 → 角色审核 → 大纲审核
       │
       ├─→ 场景细纲 → 正文生成 → 审校修订
       │
       ▼
┌─────────────┐
│   导出发布  │ ← 多格式导出
└─────────────┘
```

### 7.2 审核流程

```
阶段完成
    │
    ▼
┌─────────────┐
│  审核确认   │ ← 查看结果、提出反馈
└──────┬──────┘
       │
       ├─→ 确认通过 → 进入下一阶段
       │
       ├─→ 提出反馈 → AI 重新生成
       │
       └─→ 手动编辑 → 保存修改
```
