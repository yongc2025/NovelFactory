# 人机协作流程设计

> 版本：1.0 | 更新日期：2026-06-12
> 本文档定义人机协作的完整工作流程。

---

## 1. 问题定义

### 1.1 当前流程

```
当前系统流程（AI 主导）：

创建项目 → AI 选题 → 人确认 → AI 世界观 → 人确认 → AI 角色 → 人确认
    → AI 大纲 → 人确认 → AI 场景 → AI 正文 → AI 审校 → 人确认 → 导出
```

**问题**：
- 人只能"确认"或"重新生成"，无法深度参与
- 缺少"人写，AI 辅助"的模式
- 审校结果无法直接应用到正文

### 1.2 作家真实工作流

```
作家的真实工作流（人机协作）：

构思阶段：人有灵感 → AI 帮助扩展 → 人筛选 → AI 细化
写作阶段：人写初稿 → AI 润色 → 人修改 → AI 检查 → 人定稿
修改阶段：AI 审校 → 人看报告 → 人/AI 修改 → AI 验证
```

---

## 2. 设计目标

### 2.1 核心目标

1. **灵活参与度**：作家可以选择参与程度
2. **无缝切换**：随时在"人写"和"AI 写"之间切换
3. **闭环修改**：审校结果直接用于修改
4. **保留控制**：作家始终有最终决定权

### 2.2 协作模式

| 模式 | 人参与度 | AI 参与度 | 适用场景 |
|------|----------|----------|----------|
| **全自动** | 低（审核） | 高（生成） | 快速产出初稿 |
| **半自动** | 中（修改） | 中（生成+润色） | 基于 AI 创作 |
| **手动** | 高（写作） | 低（检查+建议） | 作家自主创作 |
| **混合** | 灵活 | 灵活 | 日常写作 |

## 3. 流程设计

### 3.1 构思阶段

#### 3.1.1 灵感扩展

```
作家输入灵感（一句话）
    │
    ▼
┌─────────────────┐
│  AI 扩展灵感    │
│  生成 3-5 个方案│
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  作家选择方案   │
│  或提出修改意见 │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  AI 细化方案    │
│  生成详细设定   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  作家确认       │
│  或继续修改     │
└─────────────────┘
```

#### 3.1.2 大纲协作

```
AI 生成大纲初稿
    │
    ▼
┌─────────────────┐
│  作家审阅大纲   │
└────────┬────────┘
         │
         ├─→ [整体确认] → 进入下一阶段
         │
         ├─→ [提出反馈] → AI 重新生成
         │
         ├─→ [手动编辑] → 打开大纲编辑器
         │
         └─→ [局部修改] → 只改某几章大纲
```

### 3.2 写作阶段

#### 3.2.1 自主写作模式

```
作家打开编辑器
    │
    ▼
┌─────────────────┐
│  作家自己写     │
│  （空白编辑器） │
└────────┬────────┘
         │
         ├─→ [写完一段] → 选中 → AI 润色
         │
         ├─→ [写不下去] → 选中要点 → AI 扩写
         │
         ├─→ [写完一章] → AI 检查
         │
         └─→ [写完保存] → 进入审校
```

#### 3.2.2 半自动写作模式

```
AI 生成初稿
    │
    ▼
┌─────────────────┐
│  作家阅读初稿   │
└────────┬────────┘
         │
         ├─→ [满意] → 确认
         │
         ├─→ [不满意段落] → 选中 → AI 改写
         │
         ├─→ [想自己改] → 打开编辑器
         │
         └─→ [整体不满意] → 提出反馈 → AI 重新生成
```

#### 3.2.3 混合写作模式

```
作家决定今天怎么写
    │
    ├─→ [想自己写] → 打开编辑器 → 自主写作模式
    │
    ├─→ [想快速产出] → AI 生成 → 半自动模式
    │
    └─→ [想改昨天的] → 打开章节 → 局部修改
```

### 3.3 修改阶段

#### 3.3.1 审校闭环

```
AI 审校章节
    │
    ▼
┌─────────────────┐
│  展示审校报告   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  作家查看问题   │
└────────┬────────┘
         │
         ├─→ [点击问题] → 跳转到对应文段
         │
         ├─→ [自动修复] → AI 针对性修改
         │
         ├─→ [手动修复] → 打开编辑器
         │
         └─→ [忽略问题] → 标记为已处理
```

#### 3.3.2 修订流程

```
审校发现问题
    │
    ▼
┌─────────────────┐
│  选择修订方式   │
└────────┬────────┘
         │
         ├─→ [AI 修订] → AI 根据审校意见修改 → 展示修改 → 确认
         │
         ├─→ [手动修订] → 打开编辑器 → 作家修改 → 保存
         │
         └─→ [混合修订] → AI 修改 → 作家微调 → 保存
```

## 4. 阶段设计

### 4.1 选题阶段

| 步骤 | 执行者 | 操作 | 输出 |
|------|--------|------|------|
| 1 | 人 | 输入灵感 | 灵感文本 |
| 2 | AI | 扩展灵感 | 3-5 个方案 |
| 3 | 人 | 选择/修改方案 | 选中方案 |
| 4 | AI | 细化方案 | 详细选题 |

**人可做的操作**：
- 选择方案
- 提出修改意见
- 手动编辑方案
- 混合修改（AI 改 + 人微调）

### 4.2 世界观阶段

| 步骤 | 执行者 | 操作 | 输出 |
|------|--------|------|------|
| 1 | AI | 基于选题生成世界观 | 世界观初稿 |
| 2 | 人 | 审阅世界观 | 确认/修改 |
| 3 | AI | 根据反馈修改 | 修改后世界观 |

**人可做的操作**：
- 确认世界观
- 提出修改意见
- 手动编辑世界观
- 添加/删除设定项

### 4.3 角色阶段

| 步骤 | 执行者 | 操作 | 输出 |
|------|--------|------|------|
| 1 | AI | 基于世界观生成角色 | 角色卡初稿 |
| 2 | 人 | 审阅角色 | 确认/修改 |
| 3 | AI | 根据反馈修改 | 修改后角色 |

**人可做的操作**：
- 确认角色
- 修改角色属性
- 添加/删除角色
- 调整角色关系

### 4.4 大纲阶段

| 步骤 | 执行者 | 操作 | 输出 |
|------|--------|------|------|
| 1 | AI | 生成完整大纲 | 大纲初稿 |
| 2 | 人 | 审阅大纲 | 确认/修改 |
| 3 | AI | 根据反馈修改 | 修改后大纲 |

**人可做的操作**：
- 确认大纲
- 修改单章大纲
- 调整章节顺序
- 添加/删除章节
- 修改伏笔设定

### 4.5 场景阶段

| 步骤 | 执行者 | 操作 | 输出 |
|------|--------|------|------|
| 1 | AI | 拆分场景 | 场景细纲 |
| 2 | 人 | 审阅场景 | 确认/修改 |
| 3 | AI | 根据反馈修改 | 修改后场景 |

**人可做的操作**：
- 确认场景
- 修改场景设定
- 调整场景顺序
- 添加/删除场景

### 4.6 正文阶段

| 步骤 | 执行者 | 操作 | 输出 |
|------|--------|------|------|
| 1 | AI/人 | 生成/写初稿 | 正文初稿 |
| 2 | 人 | 审阅/修改 | 修改后正文 |
| 3 | AI | 润色/检查 | 最终正文 |

**人可做的操作**：
- AI 生成初稿
- 自己写初稿
- 修改 AI 初稿
- 让 AI 润色
- 让 AI 扩写
- 让 AI 检查
- 手动修改

### 4.7 审校阶段

| 步骤 | 执行者 | 操作 | 输出 |
|------|--------|------|------|
| 1 | AI | 审校正文 | 审校报告 |
| 2 | 人 | 查看报告 | 确认/修改 |
| 3 | AI/人 | 修复问题 | 修改后正文 |
| 4 | AI | 验证修改 | 验证报告 |

**人可做的操作**：
- 查看审校报告
- 跳转到问题位置
- 让 AI 自动修复
- 手动修复
- 忽略问题

### 4.8 导出阶段

| 步骤 | 执行者 | 操作 | 输出 |
|------|--------|------|------|
| 1 | 人 | 选择格式 | 格式选择 |
| 2 | AI | 生成导出文件 | 导出文件 |
| 3 | 人 | 下载文件 | 最终文件 |

## 5. 操作权限设计

### 5.1 每个阶段的操作权限

| 阶段 | 人可做的操作 |
|------|--------------|
| 选题 | 选择、修改、手动编辑、混合修改 |
| 世界观 | 确认、修改、手动编辑、添加/删除 |
| 角色 | 确认、修改、添加/删除、调整关系 |
| 大纲 | 确认、修改单章、调整顺序、添加/删除 |
| 场景 | 确认、修改、调整顺序、添加/删除 |
| 正文 | AI生成、自己写、修改、润色、扩写、检查 |
| 审校 | 查看报告、跳转问题、自动修复、手动修复、忽略 |
| 导出 | 选择格式、下载 |

### 5.2 操作确认机制

```
每个阶段完成后：

┌─────────────────┐
│  阶段完成       │
│  展示结果       │
└────────┬────────┘
         │
         ├─→ [确认] → 进入下一阶段
         │
         ├─→ [修改] → 打开修改界面
         │
         ├─→ [重新生成] → AI 重新生成
         │
         └─→ [回退] → 返回上一阶段
```

## 7. 技术实现

### 7.1 前端组件

```
src/components/collaboration/
├── CollaborationMode.vue      # 协作模式选择
├── StageAction.vue            # 阶段操作栏
├── OperationHistory.vue       # 操作历史
└── ConfirmDialog.vue          # 确认对话框
```

### 7.2 API 接口

| 方法 | 端点 | 描述 |
|------|------|------|
| POST | /api/projects/{id}/stages/{stage}/confirm | 确认阶段 |
| POST | /api/projects/{id}/stages/{stage}/regenerate | 重新生成 |
| POST | /api/projects/{id}/stages/{stage}/feedback | 提出反馈 |
| GET | /api/projects/{id}/operations | 获取操作历史 |

### 7.3 状态管理

```python
# 阶段状态
class StageState:
    stage: str
    status: str  # pending/running/completed/confirmed
    result: dict
    feedback: str
    operations: list[Operation]

# 操作记录
class Operation:
    timestamp: datetime
    actor: str  # human/ai
    action: str  # generate/modify/polish/check
    target: str  # stage/scene/paragraph
    details: dict
```
