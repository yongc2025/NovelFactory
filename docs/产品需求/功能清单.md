## F04: 审校修订

### 功能描述

六线审计 + 自动修订。

### 实现详情

| 子功能 | 状态 | 代码位置 |
|--------|------|----------|
| 六线审计 | ✅ | `engine/editor.py` |
| 结构化报告 | ✅ | `engine/editor.py` |
| 自动修订 | ⚠️ 函数存在 | `engine/writer.py:revise_with_feedback` |
| 修订闭环 | ❌ 未实现 | pipeline 未调用修订函数 |

### 问题说明

**现状**：审校函数 `review_chapter` 已实现，但 pipeline 中审校后未触发修订流程。

**缺失**：
- pipeline 中审校完成后应自动调用 `revise_with_feedback`
- 修订后应重新审校验证
- 需支持用户手动反馈后修订

### 验收标准

- [x] 支持逐章审校
- [x] 生成结构化审校报告
- [ ] 支持根据审校意见自动修订
- [ ] 支持用户手动反馈修订

### API 端点

```
GET  /api/projects/{id}/review/{num}         # 获取审校报告
POST /api/projects/{id}/chapters/{num}/revise  # 修订正文（已定义，逻辑待完善）
```
