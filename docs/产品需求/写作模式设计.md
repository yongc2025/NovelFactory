# 写作模式设计

> 版本：1.0 | 更新日期：2026-06-12
> 本文档定义"人机协作写作"模式的设计方案。

---

## 1. 问题定义

### 1.1 当前问题

**现状**：系统定位是"AI 写，人审核"

```
当前流程：
  AI 生成初稿 → 人审核 → 确认/重新生成
```

**问题**：
- 作家无法自己写，只能审核 AI 的作品
- 缺少"人写，AI 辅助"的协作模式
- 无法发挥作家的创作主体性

### 1.2 作家真实需求

```
作家需要的流程：
  人写初稿 → AI 润色 → 人修改 → AI 检查 → 人定稿
  或者
  AI 生成初稿 → 人修改 → AI 润色 → 人定稿
```

---

## 2. 设计目标

### 2.1 核心目标

1. **支持作家自己写**：提供在线编辑器
2. **AI 辅助写作**：实时润色、扩写、检查
3. **人机协作**：灵活切换"人写"和"AI 写"模式

### 2.2 设计原则

| 原则 | 说明 |
|------|------|
| 作家主体 | 作家是创作者，AI 是助手 |
| 灵活切换 | 随时在"自己写"和"AI 写"之间切换 |
| 实时反馈 | 写的时候就给出建议和检查 |
| 保留控制 | 作家决定采纳或忽略 AI 建议 |

## 3. 功能设计

### 3.1 写作模式分类

| 模式 | 描述 | 适用场景 |
|------|------|----------|
| **全自动** | AI 生成，人审核 | 快速产出初稿 |
| **半自动** | AI 生成，人修改 | 基于 AI 初稿创作 |
| **手动** | 人写，AI 辅助 | 作家自主创作 |
| **混合** | 灵活切换 | 日常写作 |

### 3.3 AI 辅助功能

#### 3.3.1 AI 润色

**触发**：选中文字 → 点击"AI 润色"或快捷键

**输入**：
- 选中的文字
- 上下文（前后段落）
- 角色信息
- 文风要求

**输出**：
- 润色后的文字
- 修改说明（改了什么、为什么改）

**UI**：
```
┌─────────────────────────────────────────┐
│  AI 润色                                │
├─────────────────────────────────────────┤
│  原文：                                 │
│  "她很难过，眼泪流了下来。"             │
│                                         │
│  润色后：                               │
│  "她攥紧了衣角，指节发白。泪水无声地    │
│  滑落，滴在手背上，凉得像冬天的雨。"    │
│                                         │
│  修改说明：                             │
│  - 用动作代替心理描写                   │
│  - 增加五感细节（触觉、温度）           │
│  - 符合禁味清单（不用"难过"等抽象词）   │
│                                         │
│  [采纳] [拒绝] [再改一次]               │
└─────────────────────────────────────────┘
```

#### 3.3.2 AI 扩写

**触发**：选中文字 → 点击"AI 扩写"或快捷键

**输入**：
- 选中的文字（通常是一句话或一个要点）
- 上下文
- 目标字数

**输出**：
- 扩写后的段落

**UI**：
```
┌─────────────────────────────────────────┐
│  AI 扩写                                │
├─────────────────────────────────────────┤
│  原文：                                 │
│  "她走进房间，看到了一封信。"           │
│                                         │
│  目标字数：[100 ▼]                      │
│                                         │
│  扩写后：                               │
│  "她推开房门，冷风从窗缝钻进来。        │
│  桌上放着一封信，信封已经泛黄，          │
│  角落卷起，像是被人反复拿起又放下。      │
│  她走过去，手指碰到信封的瞬间，          │
│  缩了一下——冰凉的。"                    │
│                                         │
│  [采纳] [拒绝] [再扩一次]               │
└─────────────────────────────────────────┘
```

#### 3.3.3 AI 检查

**触发**：点击"AI 检查"或快捷键

**检查维度**：
- 角色名一致性
- 禁味清单违规
- 情节连贯性
- 情绪节奏
- 伏笔状态

**输出**：
- 问题列表（带定位）
- 修改建议

**UI**：
```
┌─────────────────────────────────────────┐
│  AI 检查报告                            │
├─────────────────────────────────────────┤
│  检查结果：85分                         │
├─────────────────────────────────────────┤
│  ⚠️ 问题1：第3段                         │
│  角色名不一致："李明"应为"张伟"         │
│  [跳转到位置] [自动修复]                │
│                                         │
│  ⚠️ 问题2：第7段                         │
│  使用了禁味词："不禁笑了"               │
│  [跳转到位置] [自动修复]                │
│                                         │
│  💡 建议：第12段                         │
│  对话信息量不足，可以增加冲突           │
│  [跳转到位置] [AI 改写]                 │
└─────────────────────────────────────────┘
```

#### 3.3.4 实时建议

**触发**：打字时实时分析

**建议类型**：
- 角色名提示（"这个角色叫什么？"）
- 禁味词警告（"建议换个词"）
- 伏笔提示（"这里可以回收之前的伏笔"）
- 情节建议（"这里可以加个冲突"）

**UI**：以气泡形式出现在光标附近

## 4. 技术实现

### 4.1 前端组件

```
src/components/editor/
├── WritingEditor.vue          # 主编辑器
├── EditorToolbar.vue          # 工具栏
├── AISuggestionPanel.vue      # AI 建议面板
├── AIRewriteDialog.vue        # AI 润色/扩写对话框
└── AICheckReport.vue          # AI 检查报告
```

### 4.2 API 接口

| 方法 | 端点 | 描述 |
|------|------|------|
| POST | /api/ai/polish | AI 润色 |
| POST | /api/ai/expand | AI 扩写 |
| POST | /api/ai/check | AI 检查 |
| POST | /api/ai/suggest | 实时建议 |

### 4.3 请求/响应格式

```json
// POST /api/ai/polish
{
  "project_id": "xxx",
  "chapter_num": 5,
  "selected_text": "她很难过，眼泪流了下来。",
  "context_before": "...",
  "context_after": "...",
  "style_requirements": "短句优先，五感写作"
}

// Response
{
  "polished_text": "她攥紧了衣角...",
  "changes": [
    {
      "type": "replace",
      "original": "她很难过",
      "replacement": "她攥紧了衣角，指节发白",
      "reason": "用动作代替心理描写"
    }
  ],
  "tokens_used": 150
}
```

    │
    ▼

         ▼
┌─────────────────┐

         │
         ▼
┌─────────────────┐
│  查看润色结果   │
│  采纳/拒绝      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  继续写作       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  完成章节       │

         │
         ▼

    ▼

         │
