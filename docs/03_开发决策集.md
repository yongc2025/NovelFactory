# 开发决策集 (Memory)

> 记录架构变更、踩坑经验与核心决策，确保 AI 在开发过程中不走弯路。

## 🏛️ 架构决策记录 (ADR)

### 1. 任务流：基于 SQLite 的 TaskRegistry
- **决策**: 引入 `TaskRegistry` 处理所有生成类后台任务，而非直接在 API 进程中运行。
- **原因**: 解决了进程重启导致的“任务失踪”问题，以及用户多次点击导致的并发冲突。目前已通过该机制支持任务排队。

### 2. 存储：JSON 文件与 SQLite 并行
- **决策**: 项目配置（World/Character等）使用 `ProjectStore` (JSON)，流水线后台状态使用 SQLite。
- **原因**: JSON 易于调试和导出，SQLite 适合管理状态。
- **计划**: 长期将全文检索 (FTS5) 纳入 SQLite，实现跨项目搜索。

### 3. LLM 路由器：Pro vs Flash
- **决策**: 定义 `Pro` 模型处理推理/逻辑密集型任务（如大纲、世界观）；`Flash` 模型处理文本密集型任务（如正文、摘要）。
- **收益**: 极大地降低了生成成本，同时保持了大框架的逻辑严密。

### 4. 提示词管理：技能驱动 (SkillLoader)
- **决策**: 废弃 `prompts.py` 中的硬编码字符串，全量迁移至 `src/novel_factory/skills/*.md`。
- **原因**: 实现了“文档即代码(Doc as Code)”，方便非开发人员（编剧/策划）直接通过修改 Markdown 来调整 AI 行为，且支持 Jinja2 复杂模板渲染。

### 5. 前端重构：模块化 Section 架构
- **决策**: 将 `ProjectDetail.vue` 拆分为多个独立的 `Section` 组件（CharacterSection, OutlineSection, etc.）。
- **原因**: 解决了 1200+ 行组件的代码堆积问题，提升了维护效率，并支持了独立的状态管理与 API 异步渲染。

### 6. 数据兼容层：API 字段规范化
- **决策**: 在 FastAPI 路由层增加 `_normalize_outline` 等中间处理函数。
- **原因**: 平滑迁移历史遗留的 JSON 数据（如 `chapter_num`）到新的前端组件期望（如 `chapter_number`），避免大规模数据刷库风险。

### 7. 状态机统一与持有化 (Task 0024)
- **决策**: 彻底废弃 FastAPI `app.py` 中的内存型 `_pipeline_states` 和 `_stage_states` 字典，改为全量映射到 SQLite `pipeline_states` 和 `stage_states` 表。
- **原因**: 解决了重启 API 后流水线状态丢失、前端“状态倒退”的体验痛点，实现了真正意义上的“断点续作”。

### 8. 伏笔追踪系统 (Task 0023)
- **决策**: 引入结构化的伏笔数据模型（`Foreshadowing`），在大纲和场景生成阶段强制要求 AI 埋设伏笔，并在正文生成时通过 `writer_skill` 动态注入待回收的伏笔信息。
- **收益**: 显著提升了长篇小说的情节密度和逻辑连贯性，避免了“写了后面忘了前面”的网文通病。

## 🧱 踩坑与避坑指南

### 1. 字符编码与 Docstring (Windows 专用)
- **踩坑**: `schema.py` 中的三引号 Docstring 包含不可见符号或不规范渲染符号时，会导致 Python 解析器报 `SyntaxError`（invalid non-printable character）。
- **规避**: 保持 Docstring 简洁且符合 UTF-8 标准，避免在三引号字符串内直接混合复杂的表格或特殊样式符号。

### 2. 数据库一致性
- **踩坑**: 手动操作数据库可能导致事务未提交或表结构不一致。
- **规避**: 使用 `db/connection.py` 提供的 Context Manager 管理连接，确保 DDL 操作在 `init_db` 时一次性完成。

### 3. 前后端契约不对齐
- **踩坑**: 后端 Pydantic 字段变动未同步到前端 TypeScript 定义，导致页面加载白屏。
- **治理**: 已确立 `0021-API 契约对齐` 任务，通过 `TaskRegistry` 统一 Response 结构，并在后端逻辑中增加字段兼容处理。同时，`PipelineStatus` 的字段源已由内存字典改为数据库视图，显著提升了可靠性。

### 3. SFC 模板错误
- **踩坑**: 在进行大规模 Vue 组件拆分或 HTML 重写时，容易出现标签未闭合或重复关闭（Invalid end tag），导致 Vite 全屏错误。
- **规避**: 每次修改后检查 `<style>` 和 `<template>` 闭合情况，确保没有残留冗余代码。

### 3. 摘要管道断裂
- **踩坑**: 之前在全量生成任务中，`writer` 模块拿不到 `prev_summary`，导致每一章都像是在写“第一章”，缺乏连贯性。
- **修复**: 在 `pipeline.py` 中强制增加摘要回写闭环。

### 4. 硬编码提示词风险
- **踩坑**: 在 `engine` 逻辑中混合业务代码和长篇 Prompt，导致代码库臃肿且难以维护。
- **规避**: **禁止**在 `engine/*.py` 中写超过 3 行的 LLM 指令。所有指令必须通过 `SkillLoader` 加载。

## 💡 关键业务逻辑
- **角色一致性**: 角色名必须与 `characters.json` 强绑定。
- **流水线互斥**: 同一个项目同时只能运行一个 P0 级任务。
