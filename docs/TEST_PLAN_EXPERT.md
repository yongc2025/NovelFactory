# NovelFactory 专家组全流程创作测试方案

> **专家组成员：**
> - **李寻欢 (首席首席内容官)**：关注故事核、价值观递进与冲突张力。
> - **梅菲斯特 (角色架构师)**：关注 12 维人物灵魂注入与角色行为逻辑的一致性。
> - **诺兰 (时空导论师)**：关注伏笔追踪 (Foreshadowing) 与故事连贯性 (Context)。
> - **张小龙 (产品交互专家)**：关注 UI 流程丝滑度、状态持久化与断点续作体验。

## 🧪 测试用例设计

### TC-01: 创作种子与流水线初始化 (李寻欢)
- **目标**：验证从输入模糊灵感（Cyberpunk + 悬疑）到生成 10 章大纲的逻辑严密性。
- **重点**：检查 `topic` 阶段的方案是否有差异化，大纲是否遵循三幕结构。

### TC-02: 12维角色灵魂注入 (梅菲斯特)
- **目标**：验证生成的角色卡是否具备“伤口”、“致命弱点”等深度属性，且在后续正文中是否被体现。
- **重点**：在正文中通过对话风格 (Speaking Style) 验证 AI 是否“入戏”。

### TC-03: 伏笔逻辑闭环 (诺兰) 
- **目标**：验证 0023 任务产出的 `foreshadow_ops`。
- **操作**：
    1. 生成大纲，检查第 1-3 章是否有 `plant`（埋设）。
    2. 检查第 8-10 章是否有与之对应的 `callback`（回收）。
    3. 生成正文，验证正文内容是否真的写出了这些细节。

### TC-04: 状态机持久化与断点续作 (张小龙)
- **目标**：验证 0024 任务产出的 SQLite 存储。
- **环节**：
    1. 启动流水线，在 `running` 状态下强杀进程（模拟崩溃）。
    2. 重启服务，验证 UI 是否能自动恢复到“正在生成”或上一个“待确认”状态。
    3. 验证“删除项目”是否干净地清理了 DB 记录。

### TC-05: 场景细纲到正文的渲染质量
- **目标**：验证“五零件法”在正文中的落地。
- **重点**：检查 `context_store` 是否正确传递了前文摘要。

---

## 📅 执行计划
1.  **环境对齐**：通过 `init_db` 确保表结构最新。
2.  **执行全路径**：从 Create -> World -> Character -> Outline -> Scene -> Draft。
3.  **压力测试**：一次性批量启动 5 章正文生成，观察 TaskRegistry 排队情况。
