## 1. 设计理念

### 1.1 核心原则：引导而非约束

**预设只用在非常确定的地方，其余全部交给用户自由发挥。**

系统的作用是"提示用户应该怎么写"，而不是"替用户决定写什么"。用户千奇百怪的脑洞是系统的核心价值，任何约束都是在降低可用性。

设计分层：

| 层 | 作用 | 覆盖率 |
|----|------|--------|
| 预设卡片 | 只在题材大类上降低冷启动门槛 | 覆盖常见题材 |
| 提示文案 | 告诉用户每步该输入什么、怎么写 | 100% |
| 自由输入 | 主路径，接纳所有脑洞 | 100% |
| AI 解析 | 理解意图，自动映射参数 | 取决于模型能力 |

### 1.2 为什么需要引导？

用户打开 NovelFactory，脑子里可能只有一个模糊的念头。让他们直接写 50-200 字的灵感描述，再填 26 个参数，就像让一个没下过厨房的人对着空白食材清单自己报菜名。

**但这不意味着要用选项框死用户。** 引导 = 告诉用户"你可以从哪些角度想" + 提供输入模板 + AI 兜底理解。

### 1.3 自由输入 vs 引导式选择

| 维度 | 自由输入（现状） | 引导+自由（方案） |
|------|-----------------|-------------------|
| **上手难度** | 高——不知道写什么 | 低——有提示告诉你从哪些角度想 |
| **创意空间** | 大——但用户容易卡住 | 大——提示打开思路，但不约束 |
| **输出质量** | 波动大 | 稳定——AI 解析兜底 |
| **适用人群** | 有经验的作者 | 所有用户 |

### 1.4 设计原则

1. **自由输入为主，预设为辅**：每个 Step 的主角是 textarea，不是卡片
2. **提示 > 选项**：告诉用户"你可以从XX角度想"，而不是"你必须选一个"
3. **AI 兜底**：不管用户输入什么格式，AI 都能理解意图并映射参数
4. **预设只用于非常确定的场景**：题材大类（8 个）、平台场景（4 个），其余不设预设
5. **永远保留出口**：引导是建议，不是强制

## 2. 引导式流程设计

### 2.0 整体流程概览

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌──────────────┐
│  Step 1     │───▶│  Step 2     │───▶│  Step 3     │───▶│  Step 4     │───▶│  Step 5     │───▶│  生成 & 审核  │
│  类型选择    │    │  核心冲突    │    │  平台+篇幅   │    │  角色快选    │    │  风格偏好    │    │  一键生成     │
│  (卡片)     │    │  (模板句)    │    │  (场景化)    │    │  (预设模板)  │    │  (参考作品)  │    │  (预览+调整)  │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘    └──────────────┘
       │                 │                  │                  │                  │
       ▼                 ▼                  ▼                  ▼                  ▼
  隐含: genre_major    隐含: premise      隐含: platforms    隐含: protagonist   隐含: tone, style
  genre_minor          world_setting      length_type        antagonist          reference_works
  target_audience      forbidden_elements target_words       supporting_count    model_provider
                                          target_chapters    has_romance
                                          chapter_word_range romance_desc
```

每步耗时预估：15-30 秒。全程 2-3 分钟完成（vs 当前 10-15 分钟）。

### 2.1 Step 1：类型选择（卡片式）

**目标**：让用户从模糊想法收敛到明确的题材方向。

#### 交互设计

- 页面展示 **6-8 张题材卡片**，横向滚动或 2×4 网格
- 每张卡片包含：图标（插画风格）、题材名称、一句话简介、热门标签
- 点击卡片选中（高亮边框），可多选（最多 2 个，主+副）
- 选中后底部出现"下一步"按钮

#### 卡片内容

| 卡片 | 图标 | 简介 | 热门标签 |
|------|------|------|----------|
| 🔄 重生复仇 | 时钟回转 | 重回过去，改写命运，手撕渣人 | 重生、打脸、逆袭、复仇 |
| 💕 甜宠恋爱 | 粉色爱心 | 甜到齁的恋爱故事，撒糖不断 | 甜宠、豪门、总裁、校园 |
| 🔮 玄幻修仙 | 仙山云海 | 修炼升级，纵横天下，逆天改命 | 修仙、升级、系统、异界 |
| 🔍 悬疑推理 | 放大镜 | 抽丝剥茧，真相只有一个 | 推理、破案、悬疑、反转 |
| 📖 都市现实 | 城市天际线 | 贴近生活的都市故事，共鸣感强 | 都市、职场、家庭、逆袭 |
| 👻 灵异恐怖 | 幽灵 | 心跳加速的惊悚体验 | 灵异、恐怖、无限流、探险 |
| 🎮 系统/游戏 | 游戏手柄 | 带着系统穿越，任务升级两不误 | 系统、游戏、数据化、穿越 |
| ✨ 古风言情 | 古扇 | 古代背景的爱恨情仇 | 古风、宫斗、宅斗、穿越 |

#### 进阶设计

- 卡片底部有"更多类型"入口，展开二级分类（如"甜宠"下可选"总裁甜宠""校园甜宠""年代甜宠"）
- 每个卡片 hover 时显示"近期爆款示例"（3 个标题），帮助用户理解该类型

#### 隐含映射

| 用户选择 | 自动映射参数 |
|----------|-------------|
| 点击"重生复仇" | `genre_major` = "重生", `genre_minor` = "复仇打脸", `target_audience` = "女频 18-35" |
| 点击"甜宠恋爱" | `genre_major` = "言情", `genre_minor` = "甜宠", `target_audience` = "女频 16-30" |
| 点击"玄幻修仙" | `genre_major` = "玄幻", `genre_minor` = "修仙升级", `target_audience` = "男频 18-35" |

### 2.2 Step 2：核心冲突（模板句引导）

**目标**：用结构化句式帮用户快速构建故事核心，替代"请描述你的灵感"。

#### 交互设计

页面分为两个区域：

**区域 A：模板句填充（推荐）**

展示一个填空模板，用户只需填写关键字段：

```
一个 [身份] 的 [主角]，
因为 [触发事件]，
不得不 [目标/任务]，
却面临 [核心阻碍]。
```

每个 `[ ]` 是一个可点击的输入框，点击后弹出预设选项卡片（可选可填）：

| 字段 | 预设示例 |
|------|---------|
| 身份 | 普通上班族、落魄千金、被陷害的天才、退役兵王、小村姑、被抛弃的弃妇… |
| 触发事件 | 重生回到十年前、意外获得系统、被渣男背叛、发现惊天秘密、被家族抛弃… |
| 目标/任务 | 复仇打脸、逆天改命、找到真相、守护家人、重回巅峰、查明死因… |
| 核心阻碍 | 强大的敌人、身世之谜、时间限制、信任危机、道德困境… |

**区域 B：快速模板（一键套用）**

对于不想填空的用户，提供 3-5 个"爆款模板"一键选择：

- 🔄 "重生复仇模板"：一个被渣男害死的女人，重生回到十年前，带着前世记忆复仇逆袭
- 💕 "甜宠模板"：一个平凡女孩意外嫁给隐藏身份的大佬，从契约婚姻到真心相爱
- 🔮 "修仙模板"：一个资质平庸的少年，意外获得上古传承，踏上逆天修炼之路
- 🔍 "悬疑模板"：一个刑警调查连环命案，发现每起案件都指向二十年前的一桩灭门惨案

**区域 C：自由输入（折叠）**

底部折叠区域，展开后是传统 textarea，供有想法的用户直接输入。

#### 选中/填写后的反馈

模板句填充完成后，在下方实时预览"故事摘要"：

> 📖 **你的故事**：一个被陷害入狱的天才律师，重生回到案发前一天，必须在 72 小时内找到真凶，洗清冤屈，却发现自己最信任的人才是幕后黑手。

#### 隐含映射

| 填写内容 | 自动映射参数 |
|----------|-------------|
| 模板句完整填写 | `premise` = 拼接后的完整描述 |
| "重生" + "复仇" | `world_setting` = "现代都市", `forbidden_elements` += ["修仙元素", "超自然力量"] |
| "意外获得系统" | `world_setting` = "现代都市+系统设定" |
| "古代"+"宫斗" | `world_setting` = "古代宫廷", `forbidden_elements` += ["现代元素"] |

### 2.3 Step 3：平台 + 篇幅（场景化描述）

**目标**：用用户能感知的"生活场景"替代抽象的参数选择。

#### 交互设计

展示 3-4 张"平台场景卡片"，每张卡片描述一个具体的创作场景：

| 卡片 | 场景描述 | 隐含参数 |
|------|---------|---------|
| 📱 小红书笔记 | "一篇 1500-3000 字的短篇，适合发笔记，一个下午就能写完" | `platforms` = ["小红书"], `length_type` = "短篇", `target_words` = 2000, `target_chapters` = 1, `chapter_word_range` = "1500-3000" |
| 📖 番茄短篇 | "10-30 章的中短篇，每天更新 2000 字，一两周写完" | `platforms` = ["番茄小说"], `length_type` = "中短篇", `target_words` = 40000, `target_chapters` = 20, `chapter_word_range` = "1500-2500" |
| 📚 番茄连载 | "50-200 章的长篇连载，每天更新 3000 字，写两三个月" | `platforms` = ["番茄小说"], `length_type` = "长篇", `target_words` = 300000, `target_chapters` = 100, `chapter_word_range` = "2500-3500" |
| 🎬 漫剧/短剧 | "改编为漫剧脚本，每集 3-5 分钟，10-20 集" | `platforms` = ["漫剧", "短剧"], `length_type` = "短篇", `target_words` = 15000, `target_chapters` = 15 |

**进阶选项**（折叠）：

选中某个场景卡片后，底部展开微调选项：
- 日更字数：1000 / 2000 / 3000 / 5000（滑块）
- 预计总字数：显示根据"日更 × 天数"计算的推荐值
- 高潮密度：低（每 5 章一个）/ 中（每 3 章一个）/ 高（每章都有钩子）

#### 隐含映射

| 场景选择 | 自动映射参数 |
|----------|-------------|
| 小红书笔记 | `climax_density` = "高", `foreshadow_count` = 0, `climax_interval` = 1 |
| 番茄连载 | `climax_density` = "中", `foreshadow_count` = 5, `climax_interval` = 3 |

### 2.4 Step 4：角色快选（预设模板 + 自定义）

**目标**：快速搭建核心角色，不需要用户自己写人物小传。

#### 交互设计

分为三个区块：

**区块 A：主角模板**

展示 6-8 张"主角人设卡片"，按 Step 1 选择的题材筛选相关人设：

| 卡片（重生复仇类） | 简介 | 隐含参数 |
|---------------------|------|---------|
| 👩‍⚖️ 被陷害的女强人 | "前世是职场精英，被闺蜜和丈夫联手陷害，重生后冷面复仇" | `protagonist_name` = "待生成", `protagonist_desc` = "重生复仇型女主，冷静果断，前世记忆加持" |
| 👨‍💼 隐忍的商业天才 | "表面是普通小职员，实际是被家族抛弃的继承人，重回巅峰" | `protagonist_desc` = "隐忍型男主，商业头脑极强，身份反转" |
| 🧑‍🎓 逆袭的学渣 | "高考落榜的学渣，重生回高三，用未来的知识碾压全场" | `protagonist_desc` = "逆袭型男主，重生加持，学业事业双线逆袭" |
| ✏️ 自定义主角 | "自己描述主角" | 展开输入框 |

**区块 B：反派/对手快选（可选）**

| 卡片 | 简介 |
|------|------|
| 🐍 白莲花闺蜜 | "表面闺蜜，暗中使坏，最终被揭穿" |
| 😈 渣男前任 | "渣男本渣，前期嚣张，后期跪舔" |
| 🏢 黑心老板 | "压榨员工的资本家，最终被反杀" |

| ⏭️ 跳过 | "不需要反派 / 后面再定" |

**区块 C：感情线选择**

用一个简单的三选一：

- 💕 有感情线（甜宠向）→ 展开：甜宠 / 虐恋 / 先婚后爱 / 暗恋成真

**区块 D：配角数量**

一个简单的选择：0 / 1-2 / 3-5 / 更多

#### 隐含映射

| 选择 | 自动映射参数 |
|------|-------------|

| "无感情线" | `has_romance` = false, `romance_desc` = "" |
| 配角"3-5" | `supporting_count` = 4 |

### 2.6 一键生成 & 审核

用户完成 5 步后，点击"✨ 一键生成我的故事方案"。

#### 生成内容

AI 根据 5 步选择，生成以下内容：

**1. 灵感描述（premise）**

> 📖 **《重生之逆袭女律师》**
>
> 林晚晴，曾经是业内最年轻的金牌律师，却在一场精心策划的陷害中被诬陷入狱。在狱中含冤而死的那一刻，她重生回到了案发前一天。
>
> 这一次，她要用 72 小时找到真凶，洗清冤屈，让每一个背叛她的人付出代价。但她很快发现，最信任的搭档才是整件事的幕后黑手……
>
> 【平台：番茄小说 | 篇幅：中短篇 20 章 | 风格：悬疑反转 + 重生复仇 | 感情线：暗恋成真】

**2. 参数预填表**

以可编辑的表格/卡片形式展示所有 26 个参数的预填值，分组展示：

| 分组 | 参数 | 预填值 | 编辑 |
|------|------|--------|------|
| 基础 | premise | (上方生成的描述) | ✏️ |
| 基础 | platforms | 番茄小说 | ✏️ |
| 基础 | length_type | 中短篇 | ✏️ |
| 题材 | genre_major | 重生 | ✏️ |
| 题材 | genre_minor | 复仇+悬疑 | ✏️ |
| ... | ... | ... | ... |

**3. AI 补充建议**

在参数表下方，AI 额外给出 2-3 条建议：

> 💡 **AI 建议**：
> - 建议加入"时间循环"元素，每次失败可以重来 72 小时，增加紧张感
> - 参考《墨雨云间》的反转节奏，每 3 章一个大反转
> - 感情线建议用"暗恋成真"，男主是检察官，既是助力也是阻碍

#### 审核交互

- **快速确认**：用户觉得没问题，直接点"✅ 确认，开始生成"→ 进入 8 阶段流水线
- **微调模式**：点击任意参数的 ✏️，展开编辑面板，修改后自动重新计算相关参数
- **重新生成**：对灵感描述不满意，点击"🔄 重新生成"，AI 基于相同参数重新写一版
- **回到引导**：点击"← 返回修改"，回到 5 步引导中的任意一步

## 4. 智能推荐逻辑

### 4.1 推荐链路

```
Step 1 (题材)
    │
    ├──▶ Step 2 推荐：根据题材推荐"核心冲突模板"
    │     例：选"重生复仇" → 推荐"被陷害→重生→复仇"模板句
    │
    ├──▶ Step 3 推荐：根据题材推荐"最适合的平台"
    │     例：选"甜宠" → 默认推荐"番茄短篇"（甜宠在番茄最火）
    │     例：选"悬疑" → 默认推荐"小红书笔记"（悬疑短篇在小红书流量好）
    │
    ├──▶ Step 4 推荐：根据题材推荐"主角/反派模板"
    │     例：选"重生复仇" → 主角推荐"被陷害的女强人"，反派推荐"白莲花闺蜜"
    │
    └──▶ Step 5 推荐：根据题材推荐"参考作品"
          例：选"重生复仇" → 推荐《重生之将门毒后》等
```

### 4.2 交叉推荐规则

| 条件 | 推荐动作 |
|------|---------|
| 题材 = 重生复仇 + 平台 = 小红书 | 自动提高 `climax_density` = "高"，缩短节奏 |
| 题材 = 甜宠 + 平台 = 番茄连载 | 推荐增加配角（丰富感情线），`supporting_count` 推荐 3-5 |
| 题材 = 悬疑 + 风格 = 暗黑 | 推荐 `foreshadow_count` = 8+，强调伏笔布局 |
| 题材 = 玄幻 + 平台 = 番茄连载 | 推荐 `target_chapters` = 100+，`climax_interval` = 2 |
| 有感情线 + 题材 = 悬疑 | 推荐感情线为辅线，不影响主线推理 |
| 选择"画面感强" + 平台 ≠ 漫剧 | 建议用户考虑漫剧/短剧改编 |

### 4.3 推荐的呈现方式

- **默认选中**：推荐选项默认高亮/勾选，用户可更改
- **推荐标签**：推荐选项旁显示"💡 推荐"标签
- **推荐理由**：hover 推荐标签时显示原因，如"甜宠题材在番茄小说流量最好"
- **不强制**：所有推荐都可覆盖，引导不等于限制

## 5. 灵活度设计

### 5.1 三层用户模型

| 用户类型 | 特征 | 使用路径 |
|----------|------|---------|
| 🟢 小白用户 | 不知道写什么，需要完全引导 | 走完 5 步引导 → 一键生成 → 确认 |
| 🟡 进阶用户 | 有想法，但需要补充细节 | 走引导但跳过部分步骤 → 微调参数 → 确认 |
| 🔴 高级用户 | 清楚自己要什么，不想被引导 | 直接点"高级模式" → 原始 26 参数表单 |

### 5.2 退出点设计

每个引导步骤都有以下出口：

1. **"跳过此步"按钮**：跳过当前步骤，使用 AI 推荐的默认值
2. **"高级模式"开关**（页面右上角）：随时切换到原始参数表单
3. **"自由输入"折叠区**：每个步骤底部都有自由输入区域
4. **"回到上一步"**：随时返回修改

### 5.3 高级模式入口

在引导流程的以下位置放置"高级模式"入口：

- 页面右上角常驻开关："⚙️ 高级模式"
- Step 1 底部："我知道我要什么，直接填参数 →"
- 生成审核页："查看全部 26 个参数 →"

高级模式 = 当前的 5 步 Tab 表单，保留原有功能，不做任何删减。

### 5.4 混合模式

允许用户在引导模式下，对某个具体步骤展开高级编辑：

```
Step 4：角色快选
```

## 7. 与后台流水线的对接

### 7.1 参数映射关系

引导步骤 → 26 个参数的完整映射：

```
Step 1 (类型选择)
  └── genre_major          ← 卡片选择
  └── genre_minor          ← 卡片选择 + 二级分类
  └── target_audience      ← 根据题材自动推导

Step 2 (核心冲突)
  └── premise              ← 模板句拼接 / 自由输入
  └── world_setting        ← 根据题材自动推导
  └── world_custom         ← 用户可选填
  └── forbidden_elements   ← 根据题材自动推导

Step 3 (平台+篇幅)
  └── platforms            ← 场景卡片选择
  └── length_type          ← 场景卡片选择
  └── target_words         ← 场景卡片选择 + 微调
  └── target_chapters      ← 场景卡片选择 + 微调
  └── chapter_word_range   ← 场景卡片选择 + 微调
  └── climax_density       ← 场景卡片选择 + 微调
  └── climax_interval      ← 根据 climax_density 推导
  └── foreshadow_count     ← 根据题材和篇幅推导

Step 4 (角色快选)
  └── protagonist_name     ← AI 生成（用户可改）
  └── protagonist_desc     ← 人设模板 / 自定义
  └── antagonist_name      ← AI 生成（用户可改）
  └── antagonist_desc      ← 人设模板 / 自定义
  └── has_romance          ← 感情线选择
  └── romance_desc         ← 感情线选择 + 细化
  └── supporting_count     ← 配角数量选择

Step 5 (风格偏好)
  └── reference_works      ← 参考作品多选
  └── tone                 ← 风格标签 + 参考作品推导
  └── style_sample         ← 用户可选填
  └── model_provider       ← 根据风格推荐（可选）

注：model_provider 在引导模式下默认使用推荐值，高级模式下可手动选择。
```

### 7.2 流水线阶段对接

8 阶段流水线如何使用引导结果：

| 流水线阶段 | 使用的参数 | 对接方式 |
|-----------|-----------|---------|
| ① 选题 | premise, genre_major, genre_minor, target_audience | 策划经理直接使用引导生成的灵感描述 |
| ② 世界观 | world_setting, world_custom, forbidden_elements | 世界观架构师基于模板展开，不需要用户从零描述 |
| ③ 角色 | protagonist_*, antagonist_*, supporting_count, has_romance, romance_desc | 角色设计师基于人设模板细化，生成完整人物小传 |
| ④ 大纲 | target_chapters, climax_density, climax_interval, foreshadow_count | 大纲编剧根据篇幅和节奏参数规划章节结构 |
| ⑤ 元数据 | platforms, length_type, tone | 内容适配师根据平台调整标题、简介、标签 |
| ⑥ 场景 | chapter_word_range, tone, style_sample | 场景编剧根据风格和字数规划场景 |
| ⑦ 正文 | model_provider, style_sample, reference_works | 正文作者根据参考作品和风格样本生成文本 |
| ⑧ 审校 | 所有参数 | 编辑审校检查是否符合用户设定的所有约束 |

### 7.3 数据流

```
用户引导输入
    │
    ▼
┌─────────────────┐
│  参数合并引擎    │  ← 合并用户选择 + 模板默认值 + AI 推荐值
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  灵感描述生成    │  ← AI 基于合并后的参数生成 50-200 字描述
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  审核 & 确认     │  ← 用户审核、微调、确认
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  ProjectConfig   │  ← 生成最终的项目配置 JSON
│  (26 params)     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  8阶段流水线     │  ← 按阶段消费参数，启动 AI 协作
└─────────────────┘
```

## 8. 实施优先级

### Phase 1：MVP（2 周）

**目标**：最小可用版本，验证引导式流程的核心假设。

#### 包含功能

- [x] Step 1：类型选择（8 张题材卡片，单选）
- [x] Step 2：核心冲突（模板句填充 + 3 个快速模板）
- [x] Step 3：平台+篇幅（4 张场景卡片）
- [x] Step 4：角色快选（主角 4 个模板 + 感情线三选一 + 配角数量）
- [x] Step 5：风格偏好（6 个标签选择 + 3 个参考作品）
- [x] 一键生成灵感描述
- [x] 参数预填表（只读展示）
- [x] "确认"按钮 → 写入 ProjectConfig
- [x] "高级模式"入口 → 跳转到原有表单

#### 不包含

- ❌ 智能推荐（使用静态默认值）
- ❌ 参数联动修改
- ❌ AI 建议
- ❌ 重新生成
- ❌ 参考作品库（使用硬编码列表）

#### 技术要点

- 引导流程为前端纯组件，不依赖后端新接口
- 最终输出与现有 ProjectConfig 格式完全兼容
- 一键生成使用现有 LLM 接口，prompt 模板化

### Phase 2：增强（2 周）

**目标**：提升引导的智能度和用户体验。

#### 包含功能

- [ ] 智能推荐：根据前面的选择推荐后续选项
- [ ] 参数联动修改：修改一个参数时自动提示相关参数
- [ ] AI 建议：生成后给出 2-3 条优化建议
- [ ] 重新生成灵感描述
- [ ] 参考作品库（动态加载，按题材分类）
- [ ] 人设模板库（更多主角/反派模板）
- [ ] 二级题材分类
- [ ] 审核页的参数编辑功能

### Phase 3：高级（2 周）

**目标**：覆盖高级用户需求，完善生态。

#### 包含功能

- [ ] 高级模式与引导模式的无缝切换（混合模式）
- [ ] 用户自定义模板保存（"我的模板"）
- [ ] 爆款分析：展示各题材/平台的近期爆款数据
- [ ] 多版本对比：生成 2-3 个方案供用户选择
- [ ] 引导流程的数据埋点和 A/B 测试
- [ ] 文风样例上传和分析
- [ ] 模板句的更多预设选项
