## 一、系统定位

### 一句话定义

> **一个人的内容工厂 —— 从一个想法，到全平台分发的多形态内容。**

### 本质

你不是在做"AI写小说工具"，你是在做**一个人的MCN内容生产线**。

```
角色：你是老板（总编/出品人），AI是你的员工（编辑部+制作部）
输入：一个选题/灵感
输出：小红书笔记 + 番茄连载 + 漫剧视频 + 短剧剧本
目标：用最少时间（每天1-2小时审核决策），让AI生产可变现的内容
```

### 差异化

```
写作猫/续写工具 → 生成一段文字，你自己拼
本系统         → 从选题到发布，全流程管线化，多形态输出
```

```
│  ─────────                                              │
```

```
│  你参与：抽检查看成品（10分钟/部）                       │
```

## 四、数据模型

### SQLite 核心表

```sql
-- 项目
CREATE TABLE projects (
    id TEXT PRIMARY KEY,
    title TEXT,
    genre TEXT,              -- 题材类型
    sub_genre TEXT,          -- 子类型
    premise TEXT,            -- 一句话前提
    target_platforms TEXT,   -- JSON: ["xiaohongshu","fanqie","comics"]
    target_words INT,        -- 目标字数
    status TEXT,             -- planning/drafting/reviewing/publishing/complete
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- 世界观设定
CREATE TABLE world_settings (
    id TEXT PRIMARY KEY,
    project_id TEXT REFERENCES projects(id),
    category TEXT,           -- time_period/rules/factions/economy/power_system
    title TEXT,
    content TEXT,
    sort_order INT
);

-- 角色
CREATE TABLE characters (
    id TEXT PRIMARY KEY,
    project_id TEXT REFERENCES projects(id),
    name TEXT,
    aliases TEXT,            -- JSON: ["小名","绰号"]
    role TEXT,               -- protagonist/antagonist/supporting
    age INT,
    appearance TEXT,         -- 外貌标记
    personality_surface TEXT,-- 表面性格
    personality_deep TEXT,   -- 深层性格
    core_desire TEXT,        -- 核心欲望
    core_fear TEXT,          -- 核心恐惧
    voice_style TEXT,        -- 说话风格
    secret TEXT,             -- 秘密
    arc_start TEXT,          -- 人物弧光起点
    arc_end TEXT,            -- 人物弧光终点
    current_state TEXT,      -- 当前状态快照
    relation_summary TEXT,   -- 与其他角色关系概述
    sort_order INT
);

-- 情节线
CREATE TABLE plot_lines (
    id TEXT PRIMARY KEY,
    project_id TEXT REFERENCES projects(id),
    name TEXT,
    plot_type TEXT,          -- main/romance/sub1/sub2
    status TEXT,             -- active/resolved/suspended
    summary TEXT,            -- 当前进展
    key_events TEXT          -- JSON: [{chapter_id, event, impact}]
);

-- 伏笔
CREATE TABLE foreshadows (
    id TEXT PRIMARY KEY,
    project_id TEXT REFERENCES projects(id),
    content TEXT,            -- 伏笔内容
    planted_chapter INT,     -- 埋设章节
    target_chapter INT,      -- 预计回收章节
    status TEXT,             -- planted/called_back/abandoned
    callback_chapter INT     -- 实际回收章节
);

-- 章节
CREATE TABLE chapters (
    id TEXT PRIMARY KEY,
    project_id TEXT REFERENCES projects(id),
    chapter_num INT,
    title TEXT,
    outline TEXT,            -- 章节大纲
    core_event TEXT,         -- 核心事件
    characters_present TEXT, -- JSON: [char_id, ...]
    emotion_position TEXT,   -- 情绪定位（铺垫/高潮/过渡）
    emotion_arc TEXT,        -- JSON: {start, peak, end}
    plot_lines_progress TEXT,-- JSON: 各情节线推进情况
    hook TEXT,               -- 章末钩子
    foreshadow_ops TEXT,     -- JSON: 伏笔操作
    status TEXT,             -- outlined/drafting/reviewed/final
    word_count INT
);

-- 场景
CREATE TABLE scenes (
    id TEXT PRIMARY KEY,
    project_id TEXT REFERENCES projects(id),
    chapter_id TEXT REFERENCES chapters(id),
    scene_num INT,
    location TEXT,
    atmosphere TEXT,
    characters_present TEXT, -- JSON
    character_goals TEXT,    -- JSON: 每个角色的目标
    conflict TEXT,
    turning_point TEXT,
    emotion_start TEXT,
    emotion_end TEXT,
    dialogue_direction TEXT, -- 对话方向（不是写对话，是写方向）
    sensory_details TEXT,    -- 五感细节提示
    detailed_plan TEXT,      -- 完整细纲
    draft TEXT,              -- 初稿正文
    final_text TEXT,         -- 定稿
    summary TEXT,            -- 自动生成的摘要
    word_count INT,
    status TEXT              -- planned/drafted/reviewed/final
);

-- 分集剧本（漫剧/短剧）
CREATE TABLE episodes (
    id TEXT PRIMARY KEY,
    project_id TEXT REFERENCES projects(id),
    episode_num INT,
    duration_seconds INT,    -- 预计时长
    script TEXT,             -- 剧本正文
    hook_first_5s TEXT,      -- 前5秒钩子
    cliffhanger TEXT,        -- 结尾悬念
    is_paid BOOLEAN,         -- 是否付费集
    storyboard TEXT,         -- JSON: 分镜脚本 [{画面描述,构图,台词,情绪}]
    image_prompts TEXT,      -- JSON: AI绘图prompt
    image_urls TEXT,         -- JSON: 生成的图片路径
    audio_url TEXT,          -- TTS音频路径
    video_url TEXT,          -- 合成视频路径
    status TEXT              -- scripted/storyboarded/illustrated/voiced/completed
);

-- 发布记录
CREATE TABLE publications (
    id TEXT PRIMARY KEY,
    project_id TEXT REFERENCES projects(id),
    platform TEXT,           -- xiaohongshu/fanqie/douyin/kuaishou/zhihu/gongzhonghao
    content_type TEXT,       -- note/chapter/episode/script
    content_id TEXT,         -- 对应的内容ID
    title TEXT,
    publish_time TIMESTAMP,
    status TEXT,             -- draft/scheduled/published
    metrics TEXT,            -- JSON: {views, likes, comments, revenue}
    notes TEXT
);
```

## 五、LLM 调用策略

### 模型分配

| 角色 | 模型 | 温度 | 理由 |
|------|------|------|------|
| 策划经理 | DeepSeek-V3 | 0.7 | 分析+创意平衡 |
| 世界观架构师 | DeepSeek-R1 | 0.7 | 复杂逻辑推演 |
| 角色设计师 | DeepSeek-V3 | 0.8 | 需要创造力 |
| 大纲编剧 | DeepSeek-R1 | 0.7 | 长链推理，结构规划 |
| 场景编剧 | DeepSeek-V3 | 0.8 | 已有大纲约束 |
| 正文作者 | DeepSeek-V3 | 0.85-0.9 | 核心创作 |
| 编辑审校 | DeepSeek-V3 | 0.1-0.3 | 精确检查 |
| 内容适配师 | DeepSeek-V3 | 0.7 | 格式转换 |
| 剧本改编编剧 | DeepSeek-V3 | 0.8 | 改编创作 |
| 漫剧制作人 | — | — | 不用LLM，用图片/TTS/视频API |

### 上下文注入策略（每次LLM调用）

```
├────────────────────────────────────┤
│ 全局记忆: 角色速查表 + 世界观摘要   │  ~1000 tokens
├────────────────────────────────────┤
│ 前文摘要: 层次化压缩               │  ~500 tokens

```
