## 一、产品定位决策

### 经讨论达成的共识

| 决策项 | 结论 | 理由 |
|--------|------|------|
| **目标平台** | 小红书 + 番茄小说 | 用户体量大，短篇门槛低 |
| **产品形态** | 先 CLI → 再 Web UI | 快速验证核心管线 |
| **MVP 用户** | 副业赚钱者 + 自媒体运营 | 用户基数大，容错空间大 |
| **核心差异化** | 长文本一致性 + 风格可定制 + 平台原生适配 | 竞品都没解决这几个问题 |
| **商业模式** | 分层订阅（免费/¥39/¥99） | 低门槛获客，专业版盈利 |

### 两个平台的本质差异

```
小红书 = 碎片化 + 视觉优先 + 3秒抓眼球 + SEO驱动
番茄   = 沉浸式 + 节奏为王 + 前3章定生死 + 完读率驱动
```

**→ 产品需要两套输出模式，不是一套模板打天下。**

## 二、技术架构方案

### 整体管线

```
[选题输入] → [世界观生成] → [大纲生成] → [细纲展开] → [逐章正文] → [校审润色] → [输出发布]
     │              │              │             │              │              │
     └──────────────┴──────────────┴─────────────┴──────────────┴──────────────┘
                              记忆系统（角色卡/情节线/伏笔/摘要）
```

### 模型分配策略

| 环节 | 模型 | 温度 | 理由 |
|------|------|------|------|
| 世界观/大纲 | DeepSeek-R1 | 0.7 | 需要逻辑推演 |
| 正文写作 | DeepSeek-V3 / MiMo | 0.85 | 需要文笔创造力 |
| 校审-规则 | DeepSeek-V3 | 0.1 | 事实核查，精确 |
| 校审-文风 | DeepSeek-V3 | 0.3 | 判断力任务 |
| 摘要/提取 | DeepSeek-V3 | 0.2 | 信息压缩 |

### 💰 成本估算（核心数据）

```
一篇 8000 字短篇 ≈ ¥0.15-0.25（约 $0.03）
一篇 10万字中篇 ≈ ¥4-5
→ 成本极低，商业模式可行！
```

### 记忆系统（解决"写着写着就崩"的核心）

```sql
-- 四张核心表
characters  -- 角色卡（姓名/性格/外貌/说话风格/当前状态）
plot_lines  -- 情节线（主线/副线/感情线，进展追踪）
foreshadows -- 伏笔（埋设场景/回收场景/状态）
scenes      -- 场景（大纲/细纲/正文/摘要/情绪弧线）
```

**上下文注入策略：分层混合**
- 全局记忆（角色卡+世界观）：~1000 tokens
- 前文摘要（层次化压缩）：~500 tokens
- 滑动窗口（最近2-3场景原文）：~2000 tokens
- 当前任务指令：~300 tokens
- 总计 ≈ 7600 tokens/调用，留足余量

## 三、创作方法论（核心武器）

### 5大爆款题材模板

| # | 题材 | 核心爽点 | 适合平台 |
|---|------|---------|---------|
| 1 | 重生/穿书 + 打脸复仇 | 信息差碾压 | 番茄为主 |
| 2 | 甜宠/先婚后爱/契约婚姻 | 假戏真做的张力 | 两平台都适合 |
| 3 | 马甲/隐藏身份/大佬下山 | 逐层揭马甲的爽感 | 番茄为主 |
| 4 | 年代文/年代重生 | 怀旧+信息差赚钱 | 番茄为主 |
| 5 | 玄学/风水/灵异+都市 | 单元剧，追读门槛低 | 两平台都适合 |

### 小红书单篇黄金结构（Beat Sheet）

```
1. 钩子开局（0-5%，150-400字）    → 😱 震惊/好奇
2. 人设立住（5-15%，300-800字）    → 🤔 代入
3. 冲突升级（15-40%，750-2000字）  → 😤 焦虑/愤怒
4. 至暗时刻（40-55%，500-1200字）  → 😢 心疼
5. 反转爆发（55-80%，750-2000字）  → 🔥 爽！
6. 情绪落点（80-100%，300-800字）  → 💖 感动/回味
```

### 番茄连载章节节奏（10章模板）

```
第1章：⚡ 开局钩子（抛出核心悬念）
第2章：🔥 小爽点（第一次打脸/反转）
第3章：📖 信息扩展（世界观展开，埋伏笔）
第4章：😤 冲突升级（反派出手）
第5章：🔥🔥 中期反转（关键信息揭露，大爽）
第6章：💕 感情线推进（CP关系升温）
第7章：😢 至暗时刻（最大危机降临）
第8章：⚡ 绝地反击（连续爽点）
第9章：🔥🔥🔥 高潮决战（终极对决）
第10章：💖 收束+彩蛋（结局+暗示续篇）
```

### 10种开头钩子公式

| # | 公式 | 示例 |
|---|------|------|
| 1 | 死亡开局 | "我死在了婚礼当天，凶手是我亲手选的新郎。" |
| 2 | 身份反差 | "全国顶级黑客的我，正在菜市场跟大妈抢打折鸡蛋。" |
| 3 | 信息炸弹 | "我花了三年才知道，我老公每月给一个女人转50万。" |
| 4 | 时间锚定+冲突 | "2024年12月31日23:58，我收到了一条来自一年前自己的短信。" |
| 5 | 直接对话 | "'你女儿的手术费，用你的肾来换。'" |
| 6 | 倒叙悬念 | "后来所有人都问我，是怎么在三天内让一个上市公司破产的。" |
| 7 | 禁忌揭露 | "整理遗物时，我在妈妈的枕头下发现了一张我从未见过的结婚证。" |
| 8 | 规则宣告 | "从今天起，谁对我好我就对谁好。谁害我，我会加倍奉还。" |
| 9 | 感官冲击 | "血从她的婚纱下摆渗出来，一滴一滴，染红了红毯。宾客们还在鼓掌。" |
| 10 | 系统激活 | "叮——检测到宿主死亡，'恶女系统'已激活。" |

### 5种章末悬念套路

1. **信息不对称** — 读者知道但角色不知道
2. **门开了/电话响了** — 新角色/新信息突然介入
3. **二选一困境** — 让读者替角色纠结
4. **打脸预告** — 预告下一章有爽点
5. **反转铺垫** — 抛出"等等，难道说……"的细节

## 四、让模型写出"网文味"的关键 Prompt

### 核心风格指令

```
1. 短句优先，一句话说一件事
2. 用动作代替心理描写（❌"她很难过" ✅"她攥紧了手机，指节发白"）
3. 对话要有信息量，不水对话
4. 五感写作（视觉/听觉/触觉），少用抽象形容词
5. 爽点用短句连击，情感段落可以放慢
6. 每1000字至少1句可截图传播的金句
7. 口语化，像朋友讲故事，不像学生写作文
```

### 禁味清单

```
❌ 不用"宛如/恍若/仿若"等文言词
❌ 不用排比句（三段式排比=作文味）
❌ 不用"不禁"
❌ 不在段落开头用"突然"
❌ 不写"心想"
❌ 不用"原来如此"类解释性语句
❌ 不让角色自己总结主题
```

### 角色卡模板（短篇精简版）

```yaml
姓名：
年龄：
外貌标记：   # 1-2个辨识度特征
核心标签：   # 3个词概括
一句话人设： # 一句话说清这个人是谁
核心欲望：
核心恐惧：
说话习惯：   # 口头禅、语气特征
致命弱点：   # 让角色立体的关键
与主角关系： # 用一个动词概括
反转点：     # 可选的秘密
```

## 五、平台运营策略

### 小红书算法关键

```
CES评分 = 点赞(1) + 收藏(1) + 评论(4) + 转发(4) + 关注(8)
流量池：200-500人初始池 → 数据达标 → 3000→1万→10万→爆款池
SEO极重要：标题/正文关键词直接影响搜索排名
```

### 番茄算法关键

```
核心指标：前3章完读率（>50%及格，>70%优秀）
冷启动期：新书3-7天初始推荐窗口
追更率：日更章节打开率
书架收藏率：>10%及格，>20%优秀
```

### 内容红线（必须内置到系统中）

```
共同红线：政治敏感、色情、暴力血腥、违法内容、未成年人相关
小红书特有：软色情零容忍、不能放微信号/外链
番茄特有：重生复仇不能涉及真实犯罪手法、不能影射真实企业/人物
```

### 安全写作技巧

```
✅ 暧昧场景用"第二天清晨"跳过
✅ 暴力用"后果描写"替代"过程描写"
✅ 敏感职业用虚构名称
✅ 涉及金钱用"一笔巨款"替代具体数额
```
