# 项目定义 — PROJECT_BRIEF.md

## 1. 目标：我要解决什么问题？

构建一个人的自动化内容工厂：从一个灵感/选题出发，通过 10 个 AI 角色协作，自动生成完整小说，并适配多平台（小红书笔记、番茄连载、漫剧视频、短剧剧本）分发形态。

核心价值：每天 1-2 小时审核决策，AI 生产可变现内容。

## 2. 现状：当前是什么情况？

**已完成：**
- ✅ 叙事引擎核心：8 阶段流水线（选题→世界观→角色→大纲→元数据→场景→正文→审校）
- ✅ 9 个 engine 模块全部实现（planner, worldbuilder, character, outliner, metadata, scene, writer, editor, memory）
- ✅ LLM 网关：DeepSeek V4 Pro/Flash + MiMo 多模型路由
- ✅ FastAPI 后端：项目 CRUD + 流水线控制 + 数据查询（~20 个端点）
- ✅ Vue 3 前端：Dashboard + 项目创建（5 级参数向导）+ 项目详情
- ✅ CLI 入口：plan / write / review / list / new 命令
- ✅ JSON 文件存储（ProjectStore）
- ✅ Markdown 导出（整体 + 分章）
- ✅ 参数化项目创建：26 个字段，5 级分层
- ✅ AI 书名/简介/标签/分类自动生成（Task 0002）
- ✅ 二次编辑功能（Task 0003）

**未完成：**
- ❌ 封面生成（AI 绘图）— Task 0004
- ❌ 发布打包（TXT/图文/视频导出）— Task 0005
- ❌ 番茄/小红书分类体系对接 — Task 0006
- ❌ 内容适配师、剧本改编编剧、漫剧制作人 3 个角色
- ❌ WebSocket 实时进度推送
- ❌ 记忆系统（层次摘要/伏笔追踪/滑动窗口）
- ❌ SQLite + FTS5 存储层
- ❌ 测试

## 3. 差距：从现状到目标，缺什么？

| 维度 | 缺口 | 优先级 |
|------|------|--------|
| 存储 | JSON 文件 → SQLite + FTS5 | P1 |
| 实时通信 | 轮询 → WebSocket 进度推送 | P1 |
| 记忆系统 | 无 → 角色状态/层次摘要/伏笔追踪/滑动窗口 | P1 |
| 导出格式 | 仅 Markdown → TXT/EPUB/PDF | P2 |
| 形态转换 | 无 → 内容适配师/剧本改编/漫剧制作 | P2 |
| 测试 | 零覆盖 → 核心流程 mock 测试 | P1 |
| 前端交互 | 基础 CRUD → 拖拽大纲/编辑模式/审校对比/日志流 | P2 |
| 发布 | 无 → 平台 API 对接 | P3 |
