## 1. 现状审计

### 1.1 当前技术栈

| 层面 | 技术 |
|------|------|
| 框架 | Vue 3.4 + TypeScript |
| 构建 | Vite 5 |
| UI 库 | Ant Design Vue 4.1 |
| 状态管理 | Pinia 2.1 |
| 样式 | Scoped CSS + Less（已安装） |
| HTTP | Axios |
| 路由 | Vue Router 4（History 模式） |

### 1.2 当前页面结构

```
/                          → Dashboard.vue（简单 Table 列表）
/projects/create           → ProjectCreate.vue（5 步 Tab 向导）
/projects/:id              → ProjectDetail.vue（侧边 Menu + 内容区）
/projects/:id/:stage       → ProjectDetail.vue（通过 route.meta.tab 切换）
```

### 1.3 现有问题诊断

| 问题 | 严重度 | 说明 |
|------|--------|------|
| **Dashboard 信息密度低** | 🔴 高 | 只有一个 Table，无统计数据、无快捷操作、无项目状态概览 |
| **创建流程枯燥** | 🔴 高 | 26 个参数平铺在 5 个 Tab 中，无引导感、无预览、无智能提示 |
| **详情页无层次** | 🟡 中 | 8 个 Tab 平铺，用户不知道当前处于流水线哪个阶段，无全局进度感 |
| **AI 感缺失** | 🟡 中 | 除了 PipelineProgress（Steps 组件）外，无任何体现"AI 编辑部"的视觉元素 |
| **组件风格原始** | 🟡 中 | 直接使用 Ant Design 默认样式，无品牌色、无自定义主题 |
| **空状态/加载状态粗糙** | 🟢 低 | 基本只有 `Spin` + `Empty`，无骨架屏、无动画 |
| **无响应式设计** | 🟢 低 | 仅 `@media (max-width: 768px)` 做了侧边栏折叠 |

## 2. 设计风格定位

### 2.1 视觉语言：「创作工坊」+「AI 智囊团」

**核心概念：** 将"AI 编辑部"具象化。不是冰冷的工具界面，而是一个有温度的创作工坊——AI 团队成员各有性格，协同帮你完成一部小说。

**设计关键词：**
- 🎨 **温暖的科技感** — 不是赛博朋克的冷蓝，而是暖色调 + 微光效果
- 📖 **文学气质** — 衬线字体点缀、纸张纹理、墨水渐变
- 🤖 **角色化 AI** — 每个流水线阶段有对应的 AI 角色头像
- ✨ **流畅的生成感** — 打字机效果、流光进度条、粒子动效

### 2.2 色彩方案

```
主色（Primary）     #6C5CE7（靛紫）  — 创意、想象力、AI 感
辅色（Secondary）   #00B894（翡翠绿）— 生长、完成、正向反馈
强调色（Accent）    #FD79A8（玫瑰粉）— 重要提示、选中状态
背景色（Background）#0F0F23（深空蓝）— 主背景（暗色模式）
表面色（Surface）   #1A1A2E（深紫灰）— 卡片/面板背景
文字色（Text）      #E2E8F0（月白）  — 主文字
次要文字            #94A3B8（银灰）  — 辅助说明
边框色              #2D2D44（暗紫灰）— 分割线、边框
成功色              #00D68F
警告色              #FFB800
错误色              #FF6B6B
```

**渐变色：**
- 主渐变：`linear-gradient(135deg, #6C5CE7, #A29BFE)` — 按钮、进度条
- AI 光晕：`radial-gradient(circle, rgba(108,92,231,0.15), transparent)` — AI 角色发光
- 流水线渐变：`linear-gradient(90deg, #6C5CE7, #00B894)` — 阶段进度

### 2.3 设计灵感来源

| 产品 | 借鉴点 |
|------|--------|
| **Notion AI** | AI 生成内容的流式展示、光标闪烁效果、内联编辑 |
| **Linear** | 极致的信息密度、暗色主题、流畅的过渡动画、键盘快捷键 |
| **Midjourney** | 生成过程的可视化、进度反馈、结果网格展示 |
| **Vercel Dashboard** | 卡片布局、数据可视化、状态标签系统 |
| **Raycast** | 命令面板、搜索体验、快速操作 |
| **Figma** | 协作感（多人光标 → AI 角色头像）、属性面板 |

### 2.4 字体方案

```css
/* 正文 */
font-family: 'Inter', 'PingFang SC', 'Microsoft YaHei', sans-serif;

/* 标题 / 文学感点缀 */
font-family: 'Playfair Display', 'Noto Serif SC', serif;

/* 代码 / 数据 */
font-family: 'JetBrains Mono', 'Fira Code', monospace;
```

从"简单项目列表"升级为"创作指挥中心"——一眼看到所有项目状态、关键指标、快捷操作。

### 3.3 核心改动

**① 统计概览卡片（新增）**
- 4 个统计卡片：总项目数 / 进行中 / 已完成 / 总产出字数
- 每个卡片有微型趋势图（sparkline）
- 卡片带微光边框效果（`box-shadow` + 渐变 border）

**② 快捷操作区（新增）**
- 3 个大按钮：新建项目 / 从模板创建 / 查看产出报告
- 按钮带 hover 动效（微微上浮 + 阴影扩散）

**③ 进行中项目（卡片视图替代 Table）**
- 每个项目一张卡片，展示：
  - 封面图 / 占位图（根据题材自动生成渐变色）
  - 项目标题 + 题材标签
  - 流水线进度条（微型，带当前阶段高亮）
  - AI 状态指示器（"AI 正在写第 3 章..." 带打字动画）
  - 最后更新时间
- 卡片 hover 效果：微微上浮 + 边框发光

**④ 全部项目（多视图切换）**
- 支持 3 种视图：卡片视图 / 列表视图 / 表格视图
- 筛选栏：状态、题材、平台、排序
- 搜索：全局搜索，支持 Cmd+K 命令面板

### 3.4 交互细节

- 新建项目按钮：hover 时 `+` 图标旋转 90° 动画
- 项目卡片：点击后有涟漪效果（ripple effect）
- 统计数字：页面加载时有从 0 到目标值的计数动画
- 空状态：无项目时展示一个大插画 + "开始你的第一部小说" 引导

## 4. 项目创建流程重设计

### 4.1 设计目标

将 26 个参数的填写从"枯燥的表单"变成"有引导感的创作对话"。

### 4.2 方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **A. 对话式引导** | 最有 AI 感，像和编辑聊天 | 实现复杂，参数多时效率低 | ⭐⭐⭐ |
| **B. 分步向导 + 预览** | 平衡引导感和效率 | 仍是表单，但体验好很多 | ⭐⭐⭐⭐⭐ |
| **C. 智能表单** | 参数分组+条件显示 | 实现中等，效果好 | ⭐⭐⭐⭐ |

**推荐方案 B：分步向导 + 实时预览**

### 4.3 新流程设计

```
Step 1: 💡 灵感输入（1 个核心字段）
  ├── 大文本框：灵感描述（必填）
  ├── AI 分析按钮：点击后 AI 自动推荐题材、角色、世界观
  └── 实时预览区：展示 AI 推荐结果

Step 2: 🎭 题材 & 平台（2-3 个字段）
  ├── 题材选择：可视化卡片选择（非下拉框）
  ├── 细分题材：联动展示
  ├── 平台选择：带平台 logo 的卡片
  └── 目标篇幅：滑块 + 预设档位

Step 3: 👥 角色设定（可跳过）
  ├── AI 自动生成的角色卡片（可编辑）
  ├── 添加自定义角色
  └── 角色关系图预览（简化版）

Step 4: 🌍 世界观（可跳过）
  ├── AI 自动生成的世界观概要（可编辑）
  ├── 关键约束条件输入
  └── 世界观可视化卡片

Step 5: ⚙️ 高级设置（可跳过）
  ├── AI 模型选择：卡片式（带模型头像+描述）
  ├── 写作风格：预设 + 自定义
  └── 生成策略：简化为 3 个档位（快/标准/精雕细琢）

Step 6: ✅ 确认 & 启动
  ├── 项目摘要卡片（所有选择的汇总）
  ├── 预估生成时间和成本
  └── 大按钮：「🚀 开始生成」
```

### 4.4 关键交互改进

**① 灵感输入框**
- 占位文字用打字机效果逐字显示示例
- 输入时右侧实时展示"AI 理解"的标签（题材、风格、基调）
- 支持语音输入（麦克风按钮）

**② 题材选择 — 从下拉框到可视化卡片**
```
┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
│  古言   │ │  现代   │ │  玄幻   │ │  科幻   │
│  🏯    │ │  🏙️    │ │  ⚔️    │ │  🚀    │
│  重生   │ │  总裁   │ │  系统流 │ │  星际   │
│  宅斗   │ │  娱乐圈 │ │  修仙   │ │  赛博   │
│  甜宠   │ │  职场   │ │  异世   │ │  时间   │
└────────┘ └────────┘ └────────┘ └────────┘
```
- 选中后展开子题材标签云
- 每个题材卡片有微动画（hover 时图标轻微浮动）

**③ 角色设定 — AI 先行**
- 进入角色 Tab 时，AI 已根据灵感自动生成 2-3 个角色
- 角色卡片可直接编辑，也可删除/新增
- 角色卡片之间有简单的"关系线"（师徒、对手、恋人等）

**④ 进度指示器**
- 顶部：步骤条（非 Tab），带完成勾选动画
- 左侧：已填参数的实时摘要
- 右侧：实时预览"项目蓝图"

### 4.5 步骤间的过渡动画

- 步骤切换：内容区向左滑出 + 新内容从右侧滑入（300ms ease）
- 完成步骤：步骤图标变为绿色勾选 + 撒花微动效
- 返回步骤：内容区向右滑出

## 5. 项目详情页重设计

### 5.1 设计目标

将 8 个流水线阶段的进展和产出以"流水线可视化"的方式呈现，让用户清楚知道"AI 编辑部"的工作进度。

### 5.2 布局方案

```

│  顶部：项目标题 + 状态标签 + 操作按钮                             │
├─────────────────────────────────────────────────────────────────┤
│  流水线进度条（横向，带 AI 角色头像）                              │
│  [💡选题] ──→ [🌍世界观] ──→ [👥角色] ──→ [📋大纲] ──→ ...     │
│   ✅完成     ✅完成      🔄AI生成中   ⏳等待                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────────────────────┐  ┌──────────────────────────────┐ │
│  │  左侧：当前阶段详情       │  │  右侧：阶段操作面板           │ │
│  │                          │  │                              │ │
│  │  📖 选题方案              │  │  ┌────────────────────────┐  │ │
│  │                          │  │  │ AI 编辑小雅 的话：     │  │ │
│  │  ┌─────────────────────┐ │  │  │                        │  │ │
│  │  │ TopicCard 1         │ │  │  │ "根据你的灵感，我推荐  │  │ │
│  │  │ (带评分、推荐标签)   │ │  │  │  重生复仇题材，目标    │  │ │
│  │  └─────────────────────┘ │  │  │  受众是 25-35 女性..." │  │ │
│  │  ┌─────────────────────┐ │  │  │                        │  │ │
│  │  │ TopicCard 2         │ │  │  │ [✅ 采用] [✏️ 修改]    │  │ │
│  │  │                     │ │  │  │ [🔄 重新生成]          │  │ │
│  │  └─────────────────────┘ │  │  └────────────────────────┘  │ │
│  │                          │  │                              │ │
│  └──────────────────────────┘  └──────────────────────────────┘ │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│  底部：阶段历史（时间线形式）                                      │
│  ● 10:30 选题方案已生成                                          │
│  ● 10:35 用户确认采用                                            │
│  ● 10:36 世界观生成中...                                         │
└─────────────────────────────────────────────────────────────────┘
```

### 5.3 流水线进度条重设计

**当前问题：** 使用 Ant Design 的 `Steps` 组件，信息量不足，无 AI 感。

**新设计：自定义流水线进度条**

```
[🧑‍💼选题员] ──→ [🏗️架构师] ──→ [👥星探] ──→ [📋编辑] ──→ [✍️作者] ──→ [🔍审校]
   ✅完成       ✅完成       🔄生成中      ⏳等待      ⏳等待      ⏳等待
   3/3 方案     已确认       角色 1/4...
```

- 每个阶段有对应的 AI 角色头像（小图标）
- 当前阶段有脉冲动画（呼吸灯效果）
- 阶段之间有连接线，已完成的线段发光
- hover 某个阶段弹出该阶段的摘要 tooltip

### 5.4 内容区重设计

**方案：双栏布局（内容 + AI 面板）**

左栏（60%）：当前阶段的产出内容
- 继续使用现有的组件（TopicCard、WorldPanel 等）
- 但增加视觉层次：卡片阴影、间距优化、信息密度提升

右栏（40%）：AI 协作面板
- AI 角色头像 + 名字（如"选题员小雅"）
- AI 的推荐理由 / 分析说明
- 操作按钮（采用 / 编辑 / 重新生成）
- 阶段时间线

### 5.5 各阶段特化设计

**选题阶段（Topic）**
- 多方案卡片并列展示（而非纵向列表）
- 每张卡片有"AI 推荐指数"雷达图
- 选中方案后有"确认采用"的大按钮动效

**世界观阶段（World）**
- 可视化世界地图 / 结构图（简化版，用 CSS 绘制）
- 关键要素用图标卡片展示（时代、地理、力量体系、社会结构）
- 约束条件用警告色标签

**角色阶段（Characters）**
- 角色卡片用"档案卡"样式（头像 + 基本属性 + 背景故事）
- 角色关系图（简化版，用 SVG 连线）
- 支持拖拽排序

**大纲阶段（Outline）**
- 时间线 / 甘特图样式的章节展示
- 每章可展开查看摘要和关键事件
- 支持拖拽调整章节顺序

**正文阶段（Chapters）**
- 阅读器模式（最大化阅读区域，减少干扰）
- 侧边栏：章节导航 + 当前章节进度
- 字数统计、阅读时间估算
- 支持"沉浸模式"（隐藏所有 UI，只保留正文）

**审校阶段（Review）**
- 数据可视化：一致性评分雷达图、问题分布饼图
- 问题列表：按严重度分组，可逐个处理
- 一键修复建议

## 6. 组件设计规范

### 6.1 间距系统（8px 基准）

```css
--space-xs: 4px;    /* 极小间距：标签内边距 */
--space-sm: 8px;    /* 小间距：元素内部 */
--space-md: 16px;   /* 中间距：卡片内边距 */
--space-lg: 24px;   /* 大间距：卡片之间 */
--space-xl: 32px;   /* 超大间距：区块之间 */
--space-2xl: 48px;  /* 特大间距：页面区块 */
```

### 6.2 圆角系统

```css
--radius-sm: 6px;   /* 按钮、标签 */
--radius-md: 10px;  /* 卡片、输入框 */
--radius-lg: 16px;  /* 大卡片、弹窗 */
--radius-xl: 24px;  /* 特殊卡片、头像 */
--radius-full: 9999px; /* 胶囊按钮 */
```

### 6.3 阴影系统

